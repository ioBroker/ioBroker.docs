# Обновление Node.js
| js контроллер | Node.js | npm |
| ------ | ----------- | ------------- |
| < 4.x | 12.x, 14.x, 16.x | 6.x |
| 4.x | 12.x, 14.x, 16.x | 6.x, 7.x, 8.x |
| 5.x | 16.x, 18.x, 20.x | 8.x, 9.x |
| **6.x** | **18.x, 20.x, 22.x** | **8.x, 9.x, 10.x** |
| **7.x** | **18.x, 20.x, 22.x, (24.x)** | **8.x, 9.x, 10.x, 11.x** |

## Зачем нужно это обновление?
ioBroker не будет работать без Node.js; подробности см. в [архитектура](https://www.iobroker.net/#de/documentation/basics/architecture.md).

Дополнительную информацию о Node.js см. в [Википедия Node.js](https://de.wikipedia.org/wiki/Node.js).

Как это часто бывает с технологиями с открытым исходным кодом, Node.js быстро развивается.

Регулярно выпускаются обновления, улучшающие **стабильность** и **безопасность**, а также добавляющие **новые функции**.

В настоящее время рекомендуемой версией для установки ioBroker является Node.js 22.x. Эта версия находится в статусе активной долгосрочной поддержки (LTS) с октября 2024 года и будет поддерживаться до апреля 2027 года.

Сообщество ioBroker официально рекомендовало эту версию, и команда `iob nodejs-update` автоматически устанавливает рекомендуемую версию.

В октябре 2025 года запланировано внедрение долгосрочной поддержки (LTS) для Node.js 24.x, после чего поддержка будет продолжаться до апреля 2028 года.

Хотя эта версия уже доступна, после завершения внедрения LTS её рекомендуется использовать только в производственных средах.

**При обновлении версий Node.js необходимо предварительно проверить и, при необходимости, исправить некоторые предварительные требования.

Важно обратить внимание на пути установки.**

**Важное уведомление, октябрь 2025 г.:** Поддержка Node.js 18.x прекращена с апреля 2025 года, и обновления безопасности больше не будут предоставляться. **В настоящее время рекомендуется использовать версию Node.js 22.x с долгосрочной поддержкой (LTS)**.

#### Рекомендуемые методы обновления
Простейший способ обновления для уже установленных ioBroker — это использование встроенной команды обновления:

```
iob nodejs-update
```

### Процедура
#### 1 - Проверьте обстоятельства
- Версия и путь
- Операционная система
- js-контроллер
- Адаптер

<details><summary>Зачем нужно проверять?</summary>

Чтобы гарантировать совместимость всех компонентов с новой версией Node.js и корректную работу установки, следует учитывать следующее: разные версии контроллеров JavaScript поддерживают разные версии Node.js, и не все адаптеры автоматически совместимы с более новыми версиями.

- Какая версия и, что наиболее важно, в какой директории находится установочный файл?
— В среде Raspberry Pi по-прежнему широко используются старые системы на базе Debian Jessie или Debian Wheezy. Для них нет версии Node.js выше 10; возможно обновление операционной системы.
— Проверьте, какая версия js-controller установлена (ее также можно увидеть на вкладке «Хост» в панели администратора). Для версий **до** js-controller 3.x, по возможности, сначала обновите js-controller. В идеале — как минимум до версии 3.2!

Например, на форуме есть вот это [Вклад](https://forum.iobroker.net/topic/42385/js-controller-3-2-jetzt-im-stable).

Во избежание несовместимостей или проблем после обновления следует проверить все адаптеры в системе и при необходимости обновить их.

Лучше всего проверить файлы README адаптера через административную панель, в журнале изменений или на GitHub соответствующего адаптера, чтобы убедиться, что установленные версии адаптера явно поддерживают запланированную версию Node.js.

</details>

#### 2 - Создание резервной копии
Перед внесением каких-либо изменений в систему необходимо создать резервную копию. В зависимости от системы доступны различные варианты. Рекомендуется использовать адаптер BackitUp или [Команда командной строки](https://www.iobroker.net/#de/documentation/config/cli.md). Резервная копия должна быть актуальной, чтобы минимизировать потерю данных.

#### 3 - Обновить адаптер
Используемые в системе адаптеры должны быть совместимы с новой версией Node.js; возможно, потребуется их обновить.

#### 4 - Остановить ioBroker
Остановить ioBroker можно с помощью собственной консольной команды или через систему управления службами.

#### 5 - Проверьте, не запущены ли еще какие-либо процессы.
Обычно это завершает все процессы. Однако, для большей безопасности, следует убедиться, что никакие процессы (адаптеры, резервные копии) больше не запущены. Также можно использовать такой инструмент, как «top», чтобы проверить, существуют ли какие-либо процессы, начинающиеся с «io.».

#### 6 - Обновление Node.js
Следующий шаг — обновление Node.js до нужной новой версии. Однако процесс обновления может отличаться в зависимости от установленной операционной системы; см. инструкции.

Менеджер пакетов Node, сокращенно `npm`, также автоматически обновляется.

#### 7 - Проверка версии и путей
После завершения обновления пути и установленные версии проверяются повторно.

#### 8 - Запустите ioBroker fixer
Поскольку установка Node.js, как упоминалось ранее, вносит некоторые изменения в систему, после этого необходимо запустить [ioBroker fixer](https://www.iobroker.net/#de/documentation/trouble/install_fixer.md). Это, помимо прочего, восстанавливает настройки безопасности, необходимые для функционирования ioBroker, а также проверяет и исправляет все разрешения.

#### 9 - Запуск ioBroker
Некоторые модули JavaScript содержат компоненты, которые необходимо скомпилировать. Этот процесс происходит во время установки. Компиляция этих модулей привязывает их к версии Node.js. После обновления эти компоненты, следовательно, необходимо перекомпилировать. Начиная с версии js-controller 3.0, предпринимается попытка обнаружить адаптеры, содержащие такие компоненты, и автоматически выполнить пересборку. Этот процесс может занять некоторое время, и затронутые адаптеры могут перезапускаться несколько раз. Это можно наблюдать в файле журнала. Проще всего это сделать в терминале с помощью команды ``iob logs --watch | uniq``.

<details><summary>Автоматическая перестройка</summary>

Начиная с js-controller 3.0, адаптеры с нативными модулями автоматически обнаруживаются, и автоматически выполняется пересборка. Этот процесс можно отслеживать в логах, и он занимает несколько минут в зависимости от системы.

ioBroker автоматически пытается определить адаптеры, которые не запускаются из-за необходимости обновления. Это работает путем распознавания типичных сообщений об ошибках и попытки соответствующего обновления. Сначала выполняется пересборка затронутого адаптера. Если это не решает проблему, обновляются зависимости адаптера. Поэтому адаптер может перезапускаться несколько раз. Пожалуйста, будьте терпеливы! Принимайте меры только в том случае, если адаптер постоянно отображается красным цветом, а в журнале указано, что пересборка не удалась.

</details>

<details><summary>Ручная пересборка</summary>

Если автоматическая пересборка не удалась, можно выполнить ручную пересборку всех адаптеров:

```bash
iobroker rebuild
```

Или же путем полной переустановки:

```bash
iobroker rebuild --install
```

Если автоматическая пересборка не удалась, её можно выполнить вручную; см. раздел «Устранение неполадок».

</details>

<details><summary>Особые случаи (например, последовательный порт)</summary>

Некоторые адаптеры, например, адаптер последовательного порта, требуют специального обращения. Возможно, потребуется их переустановка вручную.

```bash
iob install serialport
```

К сожалению, существуют особые случаи, когда даже вышеперечисленные варианты не позволяют завершить перестройку; одним из таких случаев является Serialport.

Журнал событий может выглядеть примерно так (даже после всех попыток перестроения).

Есть и другие сообщения об ошибках, но все они сводятся к одному и тому же.

Простейший вариант — пересобрать проект вручную в **правильном** каталоге.

В этом случае найдите каталог, содержащий "bindings" — выше это */opt/iobroker/node_modules/serialport/node_modules/bindings...*. В более новых версиях это может выглядеть примерно так: */opt/iobroker/node_modules/serialport/node_modules/@serialport/bindings*.

Затем перейдите в этот каталог и выполните команду `npm install --omit=dev`. После этого перезапустите адаптер.

Другой случай связан с адаптерами, имеющими модуль Canvas (например, echarts или Mihome-vacuum), где могут возникнуть проблемы.

</details>

## Инструкции для Debian/Ubuntu
#### 1 - Проверьте версию и путь
```
sudo ln -s /usr/bin/node /usr/bin/nodejs &> /dev/null
type -p nodejs node npm npx corepack && nodejs -v && node -v && npm -v && npx -v && corepack -v
```

- Версия

```
/usr/bin/nodejs
/usr/bin/node
/usr/bin/npm
/usr/bin/npx
/usr/bin/corepack
v22.x.x
v22.x.x
10.x.x
10.x.x
0.x.x
```

Важно: nodejs находится в /usr/bin, node — в /usr/bin, npm — в /usr/bin, npx — в /usr/bin, corepack — в /usr/bin, а номера версий nodejs и node, а также npm и npx, идентичны.

#### 2 - Резервное копирование
```
iobroker backup
```

- альтернативные [опции](https://www.iobroker.net/#de/documentation/config/backup.md)

#### 3 - Обновить адаптер
Инструкции можно найти в разделе [Управление адаптерами](https://www.iobroker.net/#de/documentation/tutorial/adapter.md)

```bash
iob update
```

#### 4 - Остановить ioBroker
```
iobroker stop
```

#### 5 - Проверка процессов ioBroker
```
ps aux | grep 'io\|PID'
```

- и

```
ps aux | grep 'backup\|PID'
```

- если процессы все еще продолжаются

```
sudo kill -9 [PROZESS-ID]
```

Или завершить все процессы ioBroker:

```bash
sudo pkill -f iobroker
sudo pkill -f "io\."
```

#### 6 - Обновление Node.JS
**Метод A: Собственная команда обновления ioBroker (рекомендуется для 2025 года)**

```bash
# Direktes Update auf empfohlene Version (22.x)
iob nodejs-update

# Oder spezifische Version
iob nodejs-update 22
```

**Метод B: Репозиторий NodeSource**

— Подробная информация о [Node.js](https://github.com/nodesource/distributions#installation-instructions)

```
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
```

- Для других версий Node.js просто замените `22` в URL-адресе на номер другой версии.

**По состоянию на октябрь 2025 года для ioBroker рекомендуется использовать версию Node.js 22.x!**

!> **Нельзя использовать нечетные версии Node.js.**

!> **Поддержка Node.js 18.x прекращена (с апреля 2025 года), и его больше не следует использовать.**

#### 7 - Проверка версии/пути
```
type -p nodejs node npm npx corepack && nodejs -v && node -v && npm -v && npx -v && corepack -v
```

#### 8 - Выполнить iobroker fixer
```
iobroker fix
```

#### 9 - Запуск ioBroker
```
iobroker start
```

## Инструкции для Windows
Обновление Node.js выполняется путем запуска установщика Windows.

**Рекомендуемые шаги для Windows (2025):**

1. Загрузите последнюю версию установщика Node.js для Windows с сайта [nodejs.org](https://nodejs.org) (версия 22.x LTS).
2. **Остановите ioBroker** через диспетчер служб или командную строку.
3. **Запустите установщик** (автоматическое обновление)
4. **Запустите ioBroker Fixer:** `iobroker fix`
5. **Запустите ioBroker**

**Альтернативный вариант для Windows:** Используйте подсистему Windows для Linux (WSL2) с Ubuntu, а затем следуйте инструкциям для Linux.

## Руководство по Docker
— Обновление Node.js обычно происходит путем обновления контейнера до новой версии образа Docker.
- Подробные инструкции и дополнительная информация о контейнере iobroker доступны по адресу [buanet](https://docs.buanet.de/).

## Решение проблем
### Ручная пересборка
— Информация по этому вопросу есть.

```
iobroker rebuild
```

— если этого недостаточно

```
iobroker rebuild --install
```

— Просто запустите это вручную в командной строке. В идеале это должно автоматизировать всё.

Эта команда выполняет полную переустановку всех модулей Node и должна решить большинство проблем после обновления Node.js.

### Распространенные проблемы (2025)
**"Nodejs установлен неправильно"**

```bash
# Lösung: Fixer ausführen
iobroker fix

# Falls das nicht hilft, Node.js neu installieren
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
```

**Ошибка NPM после обновления**

```bash
# NPM Cache leeren
npm cache clean --force

# Node modules neu installieren
cd /opt/iobroker
rm -rf node_modules
npm install
```

**Адаптеры не работают после обновления**

```bash
# Einzelne Adapter neu installieren
iob install [ADAPTER-NAME]

# Oder alle Adapter rebuilden
iobroker rebuild --install
```

**Проблемы с правами доступа**

```bash
# Berechtigungen korrigieren (Linux)
sudo chown -R iobroker:iobroker /opt/iobroker
iobroker fix
```

### Диагностические команды
```bash
# Umfassende Systemdiagnose
iob diag

# Node.js Installation prüfen
which node
which npm
ls -la $(which node)
node -v
npm -v

# ioBroker Status
iob status
iob list instances
```

Пока версия контроллера JavaScript ниже 4, даже при обновлении Node.js в рамках основной версии, необходимо выполнить команду `[ioBroker fixer](https://www.iobroker.net/#de/documentation/install/linux.md)`.

В будущей версии контроллера JavaScript 4 пересборка будет выполняться полностью автоматически.
