### Обновления адаптера
#### Обновления адаптера публикации
**Вопрос:** В каких файлах мне нужно изменить номер версии?

**Ответ:** В основном, вам нужно изменить 3 файла:

* `io-package.json`: изменить номер версии и добавить список последних изменений.
* `package.json`: изменение только номера версии
* `README.md`: добавлен новый номер версии и список изменений.

Обратите внимание, что необходимо использовать [Семантическое версионирование](https://semver.org/), см. [Версионирование]](https://github.com/ioBroker/ioBroker.docs/blob/master/docs/en/dev/adapterdev.md#versioning).<br> (25 ноября 2020 г.)

**Вопрос:** Мой адаптер находится в последней версии репозитория. Я обновил адаптер на Github и опубликовал его в NPM. Когда пользователи увидят новую версию в админке?

**Ответ:** ioBroker сканирует систему на наличие изменений версий дважды в день.<br> (25 ноября 2020 г.)

**Вопрос:** Как добавить новый адаптер в последнюю версию репозитория?

**Ответ:** См. [Добавьте новый адаптер в последний репозиторий.](https://github.com/ioBroker/ioBroker.repositories#add-a-new-adapter-to-the-latest-repository)<br> (25 ноября 2020 г.)

### Пользовательский интерфейс настройки адаптера (admin/index_m.html)
#### Проверка входных данных
**Вопрос:** Я хотел бы проверить поля конфигурации адаптера, используя как основные методы адаптера, так и классы/методы кода адаптера Node.js. Проверка должна происходить после того, как пользователь нажмет кнопку «Сохранить» в конфигурации адаптера, после чего будет вызван метод `save()` из `admin/index_m.html`.

**Ответ:** Вы можете использовать метод `sendTo()` для отправки переменной `obj` из `admin/index_m.html` в код адаптера, проверить ее содержимое там, а затем предоставить результат через обратный вызов методу `sendTo()` из `admin/index_m.html`.<br> Пример: Это реализовано в адаптере [План движения](https://github.com/gaudes/ioBroker.fahrplan).<br> ПРИМЕЧАНИЕ: Возможно, вам потребуется изменить ваш `io-package.json`, см., например, [ioBroker-Forum: функция sendTo() не работает](https://forum.iobroker.net/topic/5205/gel%C3%B6st-sendto-in-eigenem-adapter-funktioniert-nicht/)<br> (24 ноября 2020 г.)

### Функции адаптера
#### Запись файлов
**Вопрос:** Адаптер должен загружать файл с помощью axios и иметь возможность записывать его в iobroker-data/files/<adapter>

**Ответ:** Вот небольшой фрагмент кода для выполнения этого действия:

```
const WebCall = await axios.get(url,{responseType: "arraybuffer"});
await Helper.Adapter.writeFileAsync(Helper.Adapter.namespace, `picture.jpg`, WebCall.data)
```

Впоследствии в логе ioBroker появилось предупреждение:<br> `writeFile will not write this file (picture.jpg) in future versions: <adapter> is not an object of type "meta"`<br> В файле io-package.json необходимо добавить объект meta.user в раздел instanceObjects:<br>

```
"instanceObjects": [
  {
    "_id": "",
    "type": "meta",
    "common": {
      "name": "User files for <Adapter>",
      "type": "meta.user"
    },
    "native": {}
  }
]
```

Подробную информацию о хранении файлов, метаобъектах и поведении при резервном копировании см. в разделе [Документация по хранению файлов](filestorage.md).<br> (09-дек-2020)
