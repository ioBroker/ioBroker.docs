## Описание
Адаптер ioBroker.poolcontrol используется для управления, мониторинга и анализа систем пулов.

Она обеспечивает автоматизацию работы насосов, систем отопления, управления солнечными и фотоэлектрическими системами, а также мониторинг, диагностику, химический анализ и оценку энергоэффективности.

## Функции
### Управление и автоматизация
- **Управление насосом**
- Режимы работы: Автоматический, Автоматический (PV), Ручной, Таймер, Выкл.
- Обнаружение ошибок (отсутствие потребления энергии, питание при выключенном состоянии, перегрузка)
- Функции безопасности (защита от замерзания, защита от перегрева)
- Приоритетное владение и координация действий помощников
- Рекомендации по мощности насосов с регулируемой скоростью вращения.
- Функции обучения для анализа поведения мощности и потока (`pump.learning.*`)

- **Контроль времени**
- До 3 свободно настраиваемых недельных временных интервалов
- Постоянные значения конфигурации
- Защита от перезаписи во время обновлений

- **Контроль за солнечной энергией**
- Пороги включения/выключения коллектора с гистерезисом
- Изменение поверхности сбора данных в реальном времени для панелей мониторинга и скриптов
- Порог срабатывания предупреждения коллектора
- Дополнительный голосовой вывод для предупреждений
- Логика автоматического сброса

- **Солнечная энергия продлена**
- Отдельное управление внешними солнечными приводами
- Пороги включения/выключения Delta
- Актуальная справочная информация о пуле сборщиков данных для панелей мониторинга и скриптов.
- Максимально допустимые значения температуры воды в бассейне
- Диагностические и причинные состояния
- Логика приоритетов и блоков
- Раздел «Статус» в `solar.extended.*`
- Изменения параметра `solar.extended.pool_temperature_source`, вносимые во время выполнения, применяются автоматически; поскольку Solar Extended использует циклический интервал проверки, вычисления, логика управления и параметр `solar.extended.collector_pool_reference_delta` могут обновляться примерно до 60 секунд.

- **Управление фотоэлектрическими системами**
- Управление насосом на основе избытка солнечной энергии и потребления домохозяйства.
- Запуск логики с использованием настраиваемых дополнительных полей
- Возможность вылета за пределы взлетно-посадочной полосы в облачную погоду.
- Игнорировать режим при достижении целевого показателя циркуляции
- Поддерживает внешние идентификаторы объектов энергии.
- Режим работы насоса: `Автоматический (PV)`

- **Управление отоплением / тепловым насосом**
- Автоматическое управление нагревательным элементом или тепловым насосом
- Настраиваемые целевые и безопасные температуры
- Дополнительная опция: предварительный запуск и перезапуск насоса
- Защита прав собственности
- Логика блокировки технического обслуживания
- Поддерживает переключаемые выходы и логические состояния.
- Внутреннее состояние и диагностика в условиях перегрева.*
- Никакой химии или солнечной логики

- **Дополнительные исполнительные механизмы**
- Управление освещением
- Дополнительные насосы
- Устройства с обратным насосом
- Автоматическое включение/выключение в зависимости от режима работы насоса.
- Проверка внешних целевых состояний
- Подходит для УФ-систем, водных объектов и вспомогательных систем.

### Мониторинг и диагностика
- **Регулирование температуры**
- До 6 датчиков:
    - поверхность
- земля / дно
    - поток
    - возвращаться
- коллекционер
- температура наружного воздуха
- Минимальные/максимальные значения за сутки
- Почасовые изменения
- Разница температур
- Отслеживание последнего действительного значения
- Мониторинг и диагностика источников загрязнения
- Логика восстановления для отсутствующих обновлений
- Оценка состояния источника

- **Время воспроизведения и распространение**
- Счетчики времени выполнения (за сегодня / всего)
- Расчет кровообращения и остаточного объема
- Самовосстановление во время выполнения
- Система напоминания о обратной промывке
- Отслеживание последнего обратного потока
- Автоматический сброс после завершения обратной промывки
- Интеграция фотоэлектрических систем для достижения целевых показателей циркуляции.

- **Интеграция датчика давления**
- Измерение давления в реальном времени
- Анализ тенденций
- Средние значения обучения
- Самообучающиеся диапазоны минимальных/максимальных значений
- Диагностические состояния
- История изменения давления и его оценка
- Поддержка внешних датчиков и PoolControl PressureBox
— Только для ознакомления (без автоматического управления)

- **Проверка системы**
- Область диагностики и отладки
- Мониторинг отдельных подсистем
- Внутренний журнал отладки
- Ручная очистка журнала
- Предназначен для анализа и устранения неполадок.

### Аналитика и выводы
- **Система статистики**
- Ежедневная/еженедельная/ежемесячная статистика
- Расчеты минимального/максимального/среднего значения
- Оценка во время выполнения
- Постоянные состояния
- HTML и JSON сводки

- **Информация о солнечной энергии**
Анализ времени работы солнечных батарей
- Расчеты эффективности
- Результаты диагностики
- Ежедневный журнал
- Вывод в формате HTML / JSON / текстовый формат
- Только для ознакомления (без контроля)

- **Аналитические материалы по фотовольтаике**
- Анализ во время выполнения
- Оценка энергоэффективности
- Расчеты экономии
- Статистика запусков и работы
- HTML/JSON-сводки

- **Аналитика бассейна**
- Анализ пула на основе правил в разделе analytics.insights.pool.*
- Считывает только существующие данные PoolControl
- Отсутствует автоматическое управление, дозирование, переключение насосов или переключение исполнительных механизмов.
- Отключено по умолчанию
- Дополнительная возможность передачи сводной информации в speech.queue
- Вывод в формате HTML / JSON / текстовый формат

- **Выходные данные, готовые к визуализации**
- Вывод структурированного текста
- HTML-вывод
- JSON-сводки
- Подходит для VIS / VIS2 / панелей мониторинга

### Химия и ИИ
- **Анализ химического состава воды**

**pH**

- Ручные или внешние источники
- Логика определения местоположения измерения
- Логика стабилизации
- Поддержка ручного перемешивания
- Автоматическая дозировка отсутствует

**TDS**

- Ручные или внешние источники
- Оценка трендов (24 ч / 7 дн / 30 дн.)
- Референтные значения
- Логика определения местоположения измерения
- Вывод в формате HTML / JSON / текстовый формат

**ОВП / Редокс**

- Оценка ОРП
- Поддержка эталонного значения pH
- Только для ознакомления
- Без контроля за содержанием хлора
- Автоматическая дозировка отсутствует

**Двухуровневая, ограниченная рамками история химии**

- Существующие значения samples_json сохраняют краткосрочную историю за 7 дней и 15 минут: максимум 672 образца и 64 КБ каждый.
- Новые внутренние состояния daily_json сохраняют компактные агрегированные данные по дням локального календаря с указанием min/max/avg/last/count: максимум 32 записи и 8 КБ каждая.
- Для трендов за 24 часа и 7 дней используется samples_json; для трендов за 30 дней используется последнее значение соответствующего дневного агрегата.
- Существующие данные о трендах за 24, 7 и 30 дней, а также текстовые/HTML/JSON отчеты сохраняют свой API и смысл.
— Ежедневные сводки дополняют, но не заменяют исходные исторические данные; достоверные устаревшие данные нормализуются во время миграции, а JSON-файлы большого размера отклоняются перед анализом.
— Необработанные долгосрочные исторические данные должны храниться в специальной базе данных ioBroker, содержащей историю/временные ряды.
- Если слишком большой файл states.jsonl уже препятствует запуску js-контроллера, его необходимо исправить извне, прежде чем PoolControl сможет запуститься.

**Инструменты для химии**

- Калькулятор pH Plus
- Калькулятор pH минус
- Калькулятор соли
- Вспомогательные средства для ручных расчетов
- Поддержка предварительного заполнения объема бассейна
- Возможность ручного изменения значения
— Тексты результатов с проверкой данных и обработкой ошибок.

-Отсутствует автоматическая дозировка химикатов

- Только для ознакомления

- **Система искусственного интеллекта**
- Прогноз погоды (Open-Meteo)
- Рекомендации по бассейнам
- Ежедневный обзор
- Отчет за выходные
- Прогноз на завтра
- Дополнительные функции вывода речи
- Отслеживание дублирующегося контекста

- **Помощь по химии**
- Интерактивная помощь в изучении химии
- Типичный выбор проблемы при строительстве бассейна
- Объяснения причин и решений
- Автоматическая дозировка отсутствует
- Нет управления устройством

- Поддержка Alexa

## Конфигурация
Настройка осуществляется через вкладки в административном интерфейсе:

- **Общая информация** → Название бассейна, размер бассейна, минимальный расход воды
- **Насос** → Мощность насоса, ограничения мощности, функции безопасности
- **Температуры** → Выбор и идентификаторы объектов датчиков
- **Управление солнечной энергией** → Пороги включения/выключения, гистерезис, порог предупреждения
- **Управление временем** → Временные окна для работы насоса
- **Речевой вывод** → Активация, интеграция с Alexa/Telegram
- **Потребление и расходы** → внешний счетчик кВт·ч, цена на электроэнергию

- Расширенный анализ эффективности фотоэлектрических и солнечных систем (расчет коэффициента полезного действия, ежедневная выгода, учет погодных условий)

## Changelog
### 1.4.2 (2026-07-01)

- Fixed monthly temperature statistics reset scheduling
  - Monthly reset no longer uses long timeouts above the Node.js/ioBroker limit
  - Added persistent monthly period tracking
  - Missed month changes after adapter downtime are detected safely
  - Monthly reset is now checked daily and executed only once per period

- Improved solar logbook logging
  - Oversized solar logbook entries are now logged as debug instead of warning
  - This avoids unnecessary warning noise for non-critical diagnostic information

### 1.4.1 (2026-06-30)

- Fixed Auto-PV holding logic for already running pumps.
- When Auto-PV already controls the pump, the current pump power is now considered for the holding decision.
- This prevents a running pump from triggering its own Auto-PV afterrun/stop cycle after startup.
- The displayed PV surplus (`photovoltaic.power_surplus_w`) remains the real remaining surplus and is not artificially adjusted.

### 1.4.0 (2026-06-29)

- Added a reset button for pump learning to quickly clear learned values after a pump replacement or incorrect learning while keeping user settings intact.
- Made the daily circulation factor writable and persistent. The adapter configuration is now only used as the initial value, allowing adjustments directly via states (e.g. VIS or HomePanel).
- Added an optional temperature-dependent circulation factor that automatically increases the required daily circulation based on a selectable temperature sensor and configurable threshold.
- Extended the existing time control with an optional interval mode. Each time window can now operate either continuously or in configurable intervals without introducing a new pump mode.
- Added new diagnostic states and multilingual status messages to improve transparency and troubleshooting for the new circulation and time control features.

### 1.3.35 (2026-06-29)

- Fixed an inconsistency in the daily circulation calculation.
- `circulation.daily_remaining` is now recalculated together with `circulation.daily_required`.
- Changing the pool size or minimum daily circulation now produces consistent values immediately after adapter restart.
- The remaining daily circulation is no longer blocked by zero flow or a stopped pump.

### 1.3.34 (2026-06-27)

- **Major stability improvement:** Completely redesigned the internal chemistry history (pH, ORP and TDS) to prevent unbounded JSON state growth. This significantly reduces the risk of oversized `states.jsonl` files and potential js-controller startup failures.
- **New two-stage history architecture:** Chemistry history now uses a compact short-term history for recent measurements together with a dedicated daily history for long-term trends. All existing 24-hour, 7-day and 30-day trend calculations and reports remain fully available.
- **Protected history storage:** Added strict limits for chemistry history sample count and JSON size. Oversized or invalid history states are now safely detected, validated and handled before being processed.
- **Daily aggregates introduced:** Added compact daily aggregates for pH, ORP and TDS containing minimum, maximum, average and last measurement together with the number of valid samples. This preserves long-term trend analysis without storing large raw histories.
- **Additional safeguards:** Added size protection for the solar logbook and debug log to prevent uncontrolled state growth.
- **Maintenance:** Updated the `@iobroker/adapter-core` dependency to the latest recommended version.
