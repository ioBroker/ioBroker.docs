## Описание
Адаптер ioBroker.poolcontrol используется для управления, мониторинга и анализа систем пулов.

Она обеспечивает автоматизацию работы насосов, систем отопления, управления солнечными и фотоэлектрическими системами, а также мониторинг, диагностику, химический анализ и оценку энергоэффективности.

## Функции
### Управление и автоматизация
- **Управление насосом**
- Режимы работы: Автоматический, Автоматический (PV), Ручной, Таймер, Выкл.
- Обнаружение ошибок (отсутствие потребления энергии, питание при выключенном состоянии, перегрузка)
- Функции безопасности (защита от замерзания, защита от перегрева)
- Приоритетное владение и координация действий помощников
- Рекомендации по мощности насосов с регулируемой скоростью вращения.
- Функции обучения для анализа поведения мощности и потока (`pump.learning.*`)

- **Контроль времени**
- До 3 свободно настраиваемых недельных временных интервалов
- Постоянные значения конфигурации
- Защита от перезаписи во время обновлений

- **Контроль за солнечной энергией**
- Пороги включения/выключения коллектора с гистерезисом
- Порог срабатывания предупреждения коллектора
- Дополнительный голосовой вывод для предупреждений
- Логика автоматического сброса

- **Солнечная энергия продлена**
- Отдельное управление внешними солнечными актуаторами
- Пороги включения/выключения Delta
- Максимально допустимые значения температуры воды в бассейне
- Диагностические и причинные состояния
- Логика приоритетов и блоков
- Раздел «Статус» в `solar.extended.*`

- **Управление фотоэлектрическими системами (начиная с версии 0.6.0)**
- Управление насосом на основе избытка солнечной энергии и потребления домохозяйства.
- Запуск логики с использованием настраиваемых дополнительных полей
- Возможность вылета за пределы взлетно-посадочной полосы в облачную погоду.
- Игнорировать режим при достижении целевого показателя циркуляции
- Поддерживает внешние идентификаторы объектов энергии.
- Режим работы насоса: `Автоматический (PV)`

- **Управление отоплением/тепловым насосом (этап тестирования)**
- Автоматическое управление нагревательным элементом или тепловым насосом
- Настраиваемые целевые и безопасные температуры
- Дополнительная опция: предварительный запуск и перезапуск насоса
- Защита прав собственности
- Логика блокировки технического обслуживания
- Поддерживает переключаемые выходы и логические состояния.
- Внутреннее состояние и диагностика в условиях перегрева.*
- Никакой химии или солнечной логики

- **Дополнительные исполнительные механизмы**
- Управление освещением
- Дополнительные насосы
- Устройства с обратным насосом
- Автоматическое включение/выключение в зависимости от режима работы насоса.
- Проверка внешних целевых состояний
- Подходит для УФ-систем, водных объектов и вспомогательных систем.

### Мониторинг и диагностика
- **Регулирование температуры**
- До 6 датчиков:
    - поверхность
- земля / дно
    - поток
    - возвращаться
- коллекционер
- температура наружного воздуха
- Минимальные/максимальные значения за сутки
- Почасовые изменения
- Разница температур
- Отслеживание последнего действительного значения
- Мониторинг и диагностика источников загрязнения
- Логика восстановления для отсутствующих обновлений
- Оценка состояния источника

- **Время выполнения и распространение**
- Счетчики времени выполнения (за сегодня / всего)
- Расчет кровообращения и остаточного объема
- Самовосстановление во время выполнения
- Система напоминания о обратной промывке
- Отслеживание последнего обратного потока
- Автоматический сброс после завершения обратной промывки
- Интеграция фотоэлектрических систем для достижения целевых показателей циркуляции.

- **Интеграция датчика давления (начиная с версии 0.7.x)**
- Измерение давления в реальном времени
- Анализ тенденций
- Средние значения обучения
- Самообучающиеся диапазоны минимальных/максимальных значений
- Диагностические состояния
- История изменения давления и его оценка
- Поддержка внешних датчиков и PoolControl PressureBox
— Только для ознакомления (без автоматического управления)

- **Проверка системы**
- Область диагностики и отладки
- Мониторинг отдельных подсистем
- Внутренний журнал отладки
- Ручная очистка журнала
- Предназначен для анализа и устранения неполадок.

### Аналитика и выводы
- **Система статистики**
- Ежедневная/еженедельная/ежемесячная статистика
- Расчеты минимального/максимального/среднего значения
- Оценка во время выполнения
- Постоянные состояния
- HTML и JSON сводки

- **Информация о солнечной энергии**
- Анализ времени работы солнечных батарей
- Расчеты эффективности
- Результаты диагностики
- Ежедневный журнал
- Вывод в формате HTML / JSON / текстовый формат
- Только для ознакомления (без контроля)

- **Аналитические материалы по фотовольтаике**
- Анализ во время выполнения
- Оценка энергоэффективности
- Расчеты экономии
- Статистика запусков и работы
- HTML/JSON-сводки

- **Выходные данные, готовые к визуализации**
- Вывод структурированного текста
- HTML-вывод
- JSON-сводки
- Подходит для VIS / VIS2 / панелей мониторинга

### Химия и ИИ
- **Анализ химического состава воды**

**pH**

- Ручные или внешние источники
- Логика определения местоположения измерения
- Логика стабилизации
- Поддержка ручного перемешивания
- Автоматическая дозировка отсутствует

**TDS**

- Ручные или внешние источники
- Оценка трендов (24 ч / 7 дн / 30 дн.)
- Референтные значения
- Логика определения местоположения измерения
- Вывод в формате HTML / JSON / текстовый формат

**ОВП / Редокс**

- Оценка ОРП
- Поддержка эталонного значения pH
- Только для ознакомления
- Без контроля за содержанием хлора
- Автоматическая дозировка отсутствует

- **Система искусственного интеллекта**
- Прогноз погоды (Open-Meteo)
- Рекомендации по бассейнам
- Ежедневный обзор
- Отчет за выходные
- Прогноз на завтра
- Дополнительные функции вывода речи
- Отслеживание дублирующегося контекста

- **Помощь по химии**
- Интерактивная помощь в изучении химии
- Типичный выбор проблемы при строительстве бассейна
- Объяснения причин и решений
- Автоматическая дозировка отсутствует
- Нет управления устройством

- **Речевой вывод**
- Поддержка Alexa
- Поддержка в Telegram
- Уведомления о работе насоса, предупреждения и показания температуры

- Система информации об адаптере
- Сезонные поздравления
- Информация о версии

## Конфигурация
Настройка осуществляется через вкладки в административном интерфейсе:

- **Общая информация** → Название бассейна, размер бассейна, минимальный расход воды
- **Насос** → Мощность насоса, ограничения мощности, функции безопасности
- **Температуры** → Выбор и идентификаторы объектов датчиков
- **Управление солнечной энергией** → Пороги включения/выключения, гистерезис, порог предупреждения
- **Управление временем** → Временные окна для работы насоса
- **Речевой вывод** → Активация, интеграция с Alexa/Telegram
- **Потребление и расходы** → внешний счетчик кВт·ч, цена на электроэнергию

## Планируемые расширения
- Расширенный анализ эффективности фотоэлектрических и солнечных систем (расчет коэффициента полезного действия, ежедневная выгода, учет погодных условий)
- Функция экспорта статистики (CSV/Excel)
- Вспомогательная программа для автоматической проверки системы.
- Собственные виджеты для VIS/VIS2 (графическая визуализация бассейна и солнечной энергии)
- Специализированные модули управления для клапанов и противоточных систем
- Интеграция дополнительных блоков датчиков (например, TempBox, PressureBox, LevelBox)
- Расширение для искусственного интеллекта и голосового помощника (ежедневный отчет по бассейну, советы, голосовые команды)

## Changelog
### 1.3.25 (2026-05-26)

- Updated README structure and feature overview
- Synchronized German and English function overviews
- Updated repository maintenance dependencies

### 1.3.24 (2026-05-26)

- Updated release-script dependencies to current versions
- Improved README and changelog structure
- Repository checker recommendations reviewed

### 1.3.23 (2026-05-26)

- Added extended temperature diagnostics for all temperature sensors:
  - last valid value
  - last valid value timestamp
  - minutes since last value
  - source status (`ok`, `warning`, `not_received`, `invalid_timestamp`)
- Added automatic recovery mechanism for stalled temperature updates
- Recovery runs only when a sensor enters warning state and uses cooldown protection
- Switched temperature helper timers to ioBroker adapter timers
- Improved visibility and troubleshooting for missing or delayed temperature updates

### 1.3.22 (2026-05-24)

- Improved ORP pH reference synchronization
- ORP helper now updates pH reference independently from ORP value processing
- Immediate update of ORP pH reference when pH enabled state or pH value changes
- Fixed missing pH reference updates when ORP evaluation was blocked, invalid or waiting for measurement conditions

### 1.3.21 (2026-05-17)

NEW: Follow-pump devices

Added a new `actuators.follow_pump_devices` area.

Up to three external devices can now automatically follow the operation of the main pump.

Typical examples:

- UV systems
- Water features
- Auxiliary filters
- Additional circulation devices

Features:

- Automatic ON when the main pump starts
- Automatic OFF when the main pump stops
- Configurable target state per device
- Validation of target states:
  - state exists
  - boolean type required
  - writable required
- Protection against invalid internal follow-pump targets
- Persistent configuration values
