## Описание
Адаптер ioBroker.poolcontrol используется для управления и мониторинга систем бассейна.
Он позволяет автоматизировать управление насосом, температурой и солнечной энергией, а также анализировать энергопотребление.

## Функции
- **Управление насосом**
- Режимы работы: Автоматический, Автоматический (PV), Ручной, Таймер, Выкл.
- Автоматическое (PV) управление насосом в зависимости от избытка фотоэлектрической энергии
- Обнаружение ошибок (отсутствие потребления энергии, питание при выключенном состоянии, перегрузка)
- Функции безопасности (защита от замерзания, защита от перегрева)
- Рекомендации по мощности насоса для насосов с регулируемой скоростью вращения.

- **Регулирование температуры**
- До 6 датчиков (поверхностный, донный, расходный, обратный, коллекторный, датчик внешней температуры)
- Суточный минимум / максимум
- Почасовая смена
- Различия (например, коллектор – воздух, поверхность – дно, поток – возврат)

- **Контроль за солнечной энергией**
- Пороги включения/выключения с гистерезисом
- Предупреждение коллектора (с автоматическим сбросом при снижении уровня на 10% ниже порогового значения)
- Возможность вывода речи при предупреждении

- **Управление отоплением/тепловым насосом (новая функция, тестовая фаза)**
- Автоматическое управление нагревательным элементом или тепловым насосом в зависимости от температуры воды в бассейне.
- Целевая температура и максимальная температура безопасности настраиваются.
- Активно только при:
- Сезон купания в бассейне в разгаре
- режим работы насоса **Автоматический**
- режим технического обслуживания не активен
- Логика приоритетов:
- Режим технического обслуживания полностью блокирует управление отоплением.
- Нагрев не мешает работе насоса в ручном или почасовом режимах.
- Настраиваемое время работы насоса после окончания нагрева
- Защита прав собственности:
- Насос отключается только в том случае, если он был ранее включен самим устройством heatHelper.
- Поддерживается:
- переключаемые розетки **или**
- логические состояния управления внешними системами отопления
- Раздел «Внутреннее состояние и диагностика» в разделе «Тепло»*
- Исключительно контроль, **никакой химии или солнечной логики**.

**Примечание:** В настоящее время эта функция находится на **этапе тестирования**.
Логика полностью реализована, но на начальном этапе её следует использовать только заинтересованным пользователям, участвующим в тестировании.

- **Управление фотоэлектрическими системами (начиная с версии 0.6.0)**
- Автоматическое управление насосом на основе выработки электроэнергии фотоэлектрическими системами и потребления домохозяйством.
- Логика запуска: избыток ≥ (номинальная мощность насоса + запас прочности)
- Возможность вылета за пределы взлетно-посадочной полосы в облачную погоду.
- Игнорировать, если достигнут целевой показатель суточного оборота
- Конфигурация осуществляется с помощью двух идентификаторов внешних объектов (power_generated_id, power_house_id).
- Новый режим работы насоса «Автоматический (PV)»

- **Контроль времени**
- До 3 свободно настраиваемых временных интервалов в неделю

- **Время выполнения и распространение**
- Подсчитывает время работы (за сегодня, всего)
- Рассчитывает суточный объем циркуляции и оставшийся объем.
- Напоминание о обратной промывке с настраиваемым интервалом (например, каждые 7 дней)
— Отображение даты последней обратной промывки.
- Автоматический сброс после завершения обратной промывки
- В режиме PV учитывается состояние циркуляции (например, «Игнорировать, когда достигнуто состояние циркуляции»).

- **Потребление и затраты**
- Оценка эффективности внешнего счетчика кВт·ч
- Ежедневное, еженедельное, ежемесячное и годовое потребление
- Расчет стоимости электроэнергии на основе настраиваемой цены.

**Примечание:** Подробную информацию о поведении значений потребления и стоимости (например, после перезапуска или при замене электросчетчика) можно найти здесь:

- [Документация (на английском языке)](https://github.com/DasBo1975/ioBroker.poolcontrol/blob/main/docs/en/help.md)
- [Документация (немецкий)](https://github.com/DasBo1975/ioBroker.poolcontrol/blob/main/docs/de/help.md)

- **Система статистики**
- Раздел `analytics.statistics.*` с ежедневными, еженедельными и ежемесячными значениями.
- Автоматический расчет минимальных, максимальных, средних значений и времени выполнения.
- Полностью сохраняемые точки данных (защита от переустановки)
- Сводные данные в формате HTML и JSON по каждому датчику и общий обзор.

- **Интеграция датчика давления (начиная с версии 0.7.x)**
- Измерение давления в фильтре в режиме реального времени
- Анализ тренда: рост / падение / стабильность
- Скользящее среднее обучения (avg_bar)
- Самообучающиеся значения минимального/максимального давления
- Диагностический текст + последнее обновление
— Отсутствие автоматического управления — исключительно информационный ресурс.
- Диапазон нормального давления настраивается пользователем.

- **Система искусственного интеллекта (начиная с версии 0.8.0)**
- Модули: Советы по погоде (Open-Meteo), советы по купанию, ежедневный обзор, отчет за выходные.
- Автоматический вывод текста с возможностью вывода речи.
- Ежечасные обновления погоды для непрерывного обновления.
- Антиспамовая система для предотвращения дублирования подсказок.

- **Прогноз на завтра (aiForecastHelper, начиная с версии 0.8.0)**
- Автоматически создает ежедневный прогноз погоды на следующий день.
- Анализ температуры, погодных условий, вероятности осадков и силы ветра.
- Формирует рекомендации по использованию бассейна на следующий день (например, закрыть укрытие, ожидается небольшое количество солнечного тепла).
- Полностью основан на событиях и требует только данных Open-Meteo из геоданных ioBroker.
- Отдельные переключатели в `ai.weather.switches.*` позволяют включать/отключать отдельные функции прогноза погоды.
Результаты сохраняются в переменной `ai.weather.outputs.forecast_text`.

- **Помощь по химии (aiChemistryHelpHelper, начиная с версии 0.8.x)**
- Интерактивная, чисто информативная помощь в области химии воды с использованием искусственного интеллекта.
- Выбор типичных проблем бассейна с помощью поля выбора (например, слишком высокий/низкий уровень pH, неэффективность хлора, зеленая/мутная вода).
- Четкое описание причин и решений в текстовом формате.
- Автоматическая дозировка отсутствует
- Нет рекомендаций по товарам
- Нет управления устройством
- Отсутствие речевого вывода (исключительно визуальная информация)
- Цель: понять причины и действовать систематически (измерение → коррекция → фильтрация → повторное измерение)
— Точки данных находятся в разделе `ai.chemistry_help.*`

- **Информационная система (начиная с версии 0.7.x)**
- Система информации об адаптере
- Сезонные поздравления (Рождество, Новый год, Пасха)
- Отображение версии установленного адаптера

- **Речевой вывод**
- Вывод через Alexa или Telegram
- Объявления о запуске/остановке насоса, ошибках или температурных порогах.

- **Проверка системы (раздел диагностики)**
- Внутренний диагностический раздел для функций отладки и мониторинга.
- Выбор области для мониторинга (например, насос, солнечная энергия, температура)
- Непрерывный журнал последних изменений
- Возможна ручная очистка журнала.

Этот раздел предназначен исключительно для анализа и устранения неполадок.
В нормальном режиме работы мониторинг должен оставаться отключенным.

## Конфигурация
Настройка осуществляется через вкладки в административном интерфейсе:

- **Общая информация** → Название бассейна, размер бассейна, минимальный расход воды
- **Насос** → Мощность насоса, ограничения мощности, функции безопасности
- **Температуры** → Выбор и идентификаторы объектов датчиков
- **Управление солнечной энергией** → Пороги включения/выключения, гистерезис, порог предупреждения
- **Управление временем** → Временные окна для работы насоса
- **Речевой вывод** → Активация, интеграция с Alexa/Telegram
- **Потребление и расходы** → внешний счетчик кВт·ч, цена на электроэнергию

## Планируемые расширения
- Расширенный анализ эффективности фотоэлектрических и солнечных систем (расчет коэффициента полезного действия, ежедневная выгода, учет погодных условий)
- Функция экспорта статистики (CSV/Excel)
- Вспомогательная программа для автоматической проверки системы.
- Собственные виджеты для VIS/VIS2 (графическая визуализация бассейна и солнечной энергии)
- Управление освещением бассейна, клапанами и системами противотока.
- Интеграция дополнительных блоков датчиков (например, TempBox, PressureBox, LevelBox)
- Расширение для искусственного интеллекта и голосового помощника (ежедневный отчет по бассейну, советы, голосовые команды)

## Changelog

### 1.2.20
Release: 11.04.2026
- (DasBo) Reduced unnecessary state writes in status and photovoltaic helpers. Summary and PV timestamps are now only updated when the functional result actually changes, making the adapter quieter without affecting existing logic.

### 1.2.19
Release: 10.04.2026
- Fixed an interaction issue between `photovoltaicHelper` and `controlHelper` where automatic follow-up pumping could be stopped unexpectedly
- photovoltaicHelper now respects controlHelper priority and no longer stops the pump while automatic follow-up pumping is active
- Fixed an issue where `controlHelper` could remain in "nachpumpen" state if the pump was stopped externally
- `photovoltaic.threshold_w` is now correctly synchronized with the instance configuration
- Changes to the PV surplus threshold in adapter settings are now reliably reflected in the corresponding read-only datapoint

### 1.2.18
Release: 07.04.2026
- Fixed persistence issue for `status.season_active` (no longer overwritten on adapter start)
- Improved persistence for frost protection settings

### 1.2.17
Release: 07.04.2026
- Fix: Resolved an issue where the pressure learning reset button did not trigger reliably. The pumpHelper4 now explicitly subscribes to its relevant internal states to ensure proper event handling.

### 1.2.15
Release: 22.03.2026
- Fix i18n usage (replace I18n.t with I18n.translate) to resolve adapter startup crash and restart loop on certain systems.
