# Адаптер ioBroker для Loxone Miniserver
*** Этот адаптер требует как минимум nodejs 4.x! ***

Извлекает всю информацию, доступную в Loxone Miniserver (и Loxone Miniserver Go) и предоставляет изменения в режиме реального времени.

## Установить
Установите этот адаптер через ioBroker Admin:

1. Откройте диалоговое окно конфигурации экземпляра.
2. Введите IP-адрес или имя хоста и HTTP-порт (по умолчанию 80) вашего Loxone Miniserver
3. Создайте нового пользователя в Loxone Miniserver (используя приложение Loxone Config), которому вы предоставляете только права на чтение и запись для всех необходимых переменных.
4. Введите имя этого пользователя и его пароль в диалоге конфигурации.
5. Сохраните конфигурацию
6. Запустите адаптер

## Конфигурация
### Miniserver Hostname / IP
Это IP-адрес или имя хоста вашего Loxone Miniserver или Miniserver Go.

### Порт Miniserver
Это порт HTTP вашего Loxone Miniserver.

По умолчанию Miniserver настроен на прослушивание через порт 80, но вы могли изменить его.

### Имя пользователя Miniserver
Укажите правильное имя пользователя для доступа к Loxone Miniserver.

По соображениям безопасности настоятельно рекомендуется использовать пользователя, отличного от «admin».

Пользователю нужен только доступ на чтение к переменным, которые вы хотите использовать из ioBroker.

### Пароль Miniserver
Укажите пароль для данного имени пользователя (см. Выше).

Помните, что этот пароль хранится в ioBroker незащищенным, поэтому не используйте пользователя «admin»!

### Синхронизировать имена
Это обновит имена в ioBroker всякий раз, когда они изменяются в конфигурации Loxone.
Если это отключено, имена будут синхронизироваться только при первом обнаружении элемента управления.

### Синхронизировать комнаты
Это заполнит перечисление enum.rooms всеми комнатами, предоставленными Loxone Miniserver, и свяжет все элементы управления.

### Синхронизировать функции
Это заполнит перечисление enum.functions всеми категориями, предоставленными Loxone Miniserver, и свяжет все элементы управления.

## Состояния
Адаптер автоматически подключается к сконфигурированному Loxone Miniserver и создает состояния для каждого состояния управления, которое он находит.

Идентификаторы состояний имеют следующий формат: `loxone.<instance>.<control>.<state>`

- `<instance>` - это индекс экземпляра адаптера ioBroker (обычно «0»)
- `<control>` - это UUID элемента управления
- `<состояние>` - это состояние внутри элемента управления (для получения дополнительной информации см. [Поддерживаемые типы элементов управления] (# support-control-types)).

Имя, указанное при настройке элемента управления в Loxone Config, будет использоваться только как его отображаемое имя в ioBroker.
Это связано с тем, что пользователь может выбрать одно и то же имя для нескольких элементов управления.

Для получения дополнительной информации об элементах управления и их состояниях также ознакомьтесь с API Loxone (особенно с файлом структуры): https://www.loxone.com/enen/kb/api/

## Контроль видимости
По умолчанию Loxone Miniserver скрывает многие элементы управления (и, следовательно, их состояния) от веб-интерфейса.

Это означает, что они также скрыты от этого адаптера ioBroker.

Чтобы убедиться, что все ваши состояния должным образом сообщаются в ioBroker, убедитесь, что у них установлен флажок «Использовать в визуализации»:

![Использовать в настройках визуализации](../../../en/adapterref/iobroker.loxone/doc/loxone-config-use-in-visualization.png)

## Глобальные штаты
В настоящее время этот адаптер предоставляет следующие глобальные состояния:

- `operatingMode`: текущий номер режима работы Loxone Miniserver
- `operatingMode-text`: текущий режим работы Loxone Miniserver в виде текста
- «восход»: количество минут после полуночи, когда солнце встает сегодня
- `sunset`: количество минут после полуночи, когда солнце садится сегодня
- `уведомления`: количество уведомлений
- «модификации»: количество модификаций
- все другие глобальные государства просто сообщаются как тексты

## Поддерживаемые типы управления
Следующие типы элементов управления в настоящее время поддерживаются этим адаптером.

За названием штата вы можете увидеть тип штата:

- `(rw)`: читабельно и доступно для записи: это состояние можно изменить с ioBroker
- `(ro)`: только для чтения: это состояние нельзя изменить с ioBroker
- `(wo)`: только для записи: значение этого состояния не сообщается этим адаптером, но его можно изменить, вызвав некоторое действие на Loxone Miniserver

### Тревога
Обеспечивается контроль охранной сигнализации.

- «вооружен» (rw) логическим состоянием (true / false) тревоги; запись `true` в это значение немедленно включит сигнализацию (без предопределенной задержки)
- `nextLevel` (ro) идентификатор следующего уровня тревоги
    * 1 = Тихо
    * 2 = Acustic
    * 3 = Оптический
    * 4 = Внутренний
    * 5 = Внешний
    * 6 = Удаленный
- `nextLevelDelay` (ro) задержка следующего уровня в секундах
- `nextLevelDelayTotal` (ro) общая задержка следующего уровня в секундах
- `level` (ro) идентификатор текущего уровня тревоги
    * 1 = Тихо
    * 2 = Acustic
    * 3 = Оптический
    * 4 = Внутренний
    * 5 = Внешний
    * 6 = Удаленный
- `startTime` (ro) временная метка при запуске тревоги
- ʻArmedDelay` (ro) задержка включения охранного управления
- ʻArmedDelayTotal` (ro) общая задержка включения охранного управления
- `Датчики` (ро) список датчиков
- `disabledMove` (rw) движение отключено (true) или нет (false)
- `delayedOn` (wo) запись любого значения в это состояние включает сигнализацию с настроенной задержкой
- `quit` (wo), записывающая любое значение в это состояние, подтверждает тревогу

### Центральная сигнализация
Предусмотрено центральное управление охранной сигнализацией.

- «вооружен» (rw) логическим состоянием (true / false) тревоги; запись `true` в это значение немедленно включит сигнализацию (без предопределенной задержки)
- `delayedOn` (wo) запись любого значения в это состояние включает сигнализацию с настроенной задержкой
- `quit` (wo), записывающая любое значение в это состояние, подтверждает тревогу

### Будильник
Предусмотрено управление будильником.

- `isEnabled` (rw) логическое состояние (true / false) будильника
- `isAlarmActive` (ro) логическое (true / false), звонит ли в данный момент сигнал тревоги
- `validNeeded` (ro) логическое (true / false), нужно ли пользователю подтверждать тревогу
- `ringingTime` (ro) обратный отсчет в секундах, как долго будильник будет звонить до тех пор, пока он снова не будет дремать
- длительность `ringDuration` (rw) в секундах, в течение которой звонит будильник
- `prepareDuration` (rw) время подготовки в секундах
- `snoozeTime` (ro) секунд до окончания дремоты
- длительность `snoozeDuration` (rw) в секундах дремоты
- `snooze` (wo) запись любого значения в это состояние откладывает текущий сигнал тревоги
- `dismiss` (wo) записывает любое значение в это состояние, отклоняет текущий аварийный сигнал.

### AudioZone
Предоставлено Music Server Zone.

- `serverState` (ro) состояние музыкального сервера:
    * -3 = неизвестная / недействительная зона
    * -2 = недоступно
    * -1 = неизвестно
    * 0 = не в сети
    * 1 = инициализация (загрузка, попытка добраться до нее)
    * 2 = онлайн
- `playState` (rw) состояние воспроизведения:
    * -1 = неизвестно (это значение не может быть установлено)
    * 0 = остановлено (установка этого значения приостановит воспроизведение)
    * 1 = приостановлено (установка этого значения приостановит воспроизведение)
    * 2 = воспроизведение (установка этого значения запустит / возобновит воспроизведение)
- `clientState` (ro) состояние клиента:
    * 0 = не в сети
    * 1 = инициализация (загрузка, попытка добраться до нее)
    * 2 = онлайн
- `power` (rw) независимо от того, активна ли мощность клиента
- `volume` (rw) текущий объем
- зонам maxVolume (ro) можно назначить максимальную громкость
- `shuffle` (rw) независимо от того, включен или нет список воспроизведения в случайном порядке
- `sourceList` (ro) список, содержащий все зоны-избранное
- `repeat` (rw) режим повтора:
    * -1 = неизвестно
    * 0 = выкл
    * 1 = повторить все
    * 2 = -не используется-
    * 3 = повторить текущий элемент
- `songName` (ro) название песни
- `duration` (ro) как долго весь трек, -1, если не известно (поток)
- `прогресс` (rw) текущая позиция в треке
- `album` (ro) название альбома
- `Artist` (ro) имя исполнителя
- `станция` (ро) название станции
- `жанр` (ро) название жанра
- `cover` (ro) песня / альбом обложка обложки URL
- `source` (rw) текущий выбранный идентификатор источника (см.` sourceList` выше)
- `prev` (wo), записывающая любое значение в это состояние, перемещается на предыдущую дорожку
- `next` (wo), записывающее любое значение в это состояние, перемещается на следующую дорожку

### Central Audio
Предоставляется центральным музыкальным сервером.

- `control` (wo) устанавливает состояние воспроизведения всех игроков (` true` = игра, `false` = пауза)

### Colorpicker
Это устройство появляется только внутри LightController.

- `red` (rw) значение красного цвета для палитры цветов
- `green` (rw) значение зеленого цвета для палитры цветов
- `blue` (rw) значение синего цвета для палитры цветов

Установка одного или нескольких из перечисленных выше состояний из ioBroker отправит команду на Miniserver только примерно через 100 мс.
Это необходимо для предотвращения многократного изменения цвета для одного пользовательского ввода.

### Colorpicker V2
Это устройство появляется только внутри Light Controller V2 в программном обеспечении Loxone версии 9 и выше.

- `red` (rw) значение красного цвета для палитры цветов
- `green` (rw) значение зеленого цвета для палитры цветов
- `blue` (rw) значение синего цвета для палитры цветов

Установка одного или нескольких из перечисленных выше состояний из ioBroker отправит команду на Miniserver только примерно через 100 мс.
Это необходимо для предотвращения многократного изменения цвета для одного пользовательского ввода.

### Диммер
Предоставлено диммерами.

- `position` (rw) текущая позиция для диммера
- `min` (ro) текущее минимальное значение
- `max` (ro) текущее максимальное значение
- `step` (ro) текущее значение шага
- `on` (wo) запись любого значения в это состояние устанавливает диммер в последнюю известную позицию
- `off` (wo) запись любого значения в это состояние отключает диммер, устанавливает позицию в 0, но запоминает последнюю позицию

### Ворота
Предусмотрено управление воротами.

- `position` (ro) позиция от 1 = до 0 = вниз
- «активное» (rw) текущее направление движения ворот
    * -1 = закрыть
    * 0 = не двигается
    * 1 = открыто
- `protectOpen` (ro), препятствующий открытию двери
- `protectClose` (ro), препятствующий закрытию двери

### Центральные ворота
Обеспечивается центральным управлением ворот.

- `open` (wo) открывает все ворота
- `close` (wo) закрывает все ворота
- `stop` (wo) останавливает все моторы ворот

### InfoOnlyDigital
Предоставляется виртуальными состояниями, а также сенсорным переключателем Loxone.

- `активное` (ro) логическое состояние (true / false) элемента управления
- `active-text` (ro), если настроено, текстовый эквивалент состояния
- `active-image` (ro), если настроено, изображение эквивалентно состоянию
- `active-color` (ro), если настроено, цветовой эквивалент состояния

![InfoOnlyDigital настройки](../../../en/adapterref/iobroker.loxone/doc/loxone-config-info-only-digital.png)

### InfoOnlyAnalog
Предоставляется виртуальными состояниями, а также сенсорным переключателем Loxone.

- `value` (ro) значение состояния (число) элемента управления
- `value-formatted` (ro), если настроено, форматированное значение состояния (с использованием формата" Unit "из Loxone Config)

### Переговорное устройство
Предоставляется дверными контроллерами.

- `колокол` (ро) звонит ли колокол
- массив `lastBellEvents` (ro), содержащий метки времени для каждого колокольного действия, на которое не было ответа
- `version` (ro) Только для Loxone Intercoms - текст, содержащий текущую установленную прошивку

версии

- `answer` (wo) запись любого значения в это состояние деактивирует звонок

Этот тип канала может содержать другие устройства. См. Соответствующую главу для получения дополнительной информации.

### Жалюзи
Предусмотрены разные виды жалюзи (автоматические и ручные).

- `up` (rw) ли Jalousie движется вверх
- `down` (rw), движется ли Jalousie вниз
- `position` (ro) позиция жалюзи, число от 0 до 1
    * Жалюзи верхнее положение = 0
    * Жалюзи нижнее положение = 1
- `shadePosition` (ro) позиция жалюзи (жалюзи), число от 0 до 1
    * Жалюзи не затенены = 0
    * Жалюзи затенены = 1
- `safetyActive` (ro) используется только теми, кто работает с автопилотом, это означает безопасное отключение
- `autoAllowed` (ro) используется только теми, кто имеет автопилот
- `autoActive` (rw) используется только теми, кто использует автопилот
- `locked` (ro) только теми, у кого есть автопилот, это представляет выходной QI в Loxone Config
- `infoText` (ro) сообщает, например, на то, что вызвало заблокированное состояние, или что вызвало активацию безопасности.
- `fullUp` (wo) записывает любое значение в это состояние, запускает полное движение
- `fullDown` (wo), записывающая любое значение в это состояние, запускает движение вниз
- `shade` (wo), записывающий любое значение в это состояние, приводит жалюзи в идеальное положение

### Центральные Жалюзи
Обеспечивается центральное управление жалюзи.

- `autoActive` (rw) используется только теми, кто использует автопилот
- `fullUp` (wo) записывает любое значение в это состояние, запускает полное движение
- `fullDown` (wo), записывающая любое значение в это состояние, запускает движение вниз
- `shade` (wo), записывающий любое значение в это состояние, оттенки всех жалюзи в идеальную позицию

### Контроллер света
Предоставляются (гостиница) контроллеры освещения.
Сцены могут быть изменены только в приложениях Loxone, но могут быть выбраны в ioBroker.

- `activeScene` (rw) номер текущей активной сцены
    * 0: все выключено
    * 1..8: пользовательская сцена (определение / изучение сцен должно выполняться с помощью инструментов Loxone)
    * 9: все включено
- `sceneList` (ro) список всех сцен
- `плюс` (wo) меняет на следующую сцену
- `минус` (wo) меняет предыдущую сцену

Этот тип канала может содержать другие устройства. См. Соответствующую главу для получения дополнительной информации.

### Light Controller V2
Предоставляется (гостиничными) контроллерами освещения в программном обеспечении Loxone версии 9 и выше.
Настроения можно изменять только в приложениях Loxone, но их можно выбирать и комбинировать в ioBroker.

- `moodList` (ro) список всех настроенных имен настроений
- `activeMoods` (rw) текущий активный список имен настроений
- `FavoritesMoods` (ro) список любимых имен настроения
- `AdditionalMoods` (ro) список нелюбимых имен настроений
- «плюс» (wo) меняется на следующее настроение
- `минус` (wo) меняет прежнее настроение

Этот тип канала может содержать другие устройства. См. Соответствующую главу для получения дополнительной информации.

### Центральный световой контроллер
Предоставляется центральным контроллером освещения.

- `control` (wo) включает или выключает все огни

### Метр
Предоставляется коммунальными счетчиками.

- `фактическое` (ро) фактическое значение (число)
- `actual-formatted` (ro), если настроено, отформатированное фактическое значение состояния (используя формат" Unit "из Loxone Config)
- `total` (ro) общее значение (число)
- `total-formatted` (ro), если настроено, форматированное общее значение состояния (используя формат" Unit "из Loxone Config)
- `reset` (wo) запись любого значения в это состояние сбрасывает общее значение

### Нажать кнопку
Обеспечивается виртуальными кнопочными входами.

- `active` (rw) текущее состояние кнопки
- `pulse` (wo), записывающий любое значение в это состояние, будет имитировать нажатие кнопки только в течение очень короткого времени.

### Слайдер
Предоставляются аналоговые виртуальные входы.

- `value` (rw) текущее значение ползунка
- `value-formatted` (ro), если настроено, форматированное значение состояния (с использованием формата" Unit "из Loxone Config)
- `error` (ro) указывает на недопустимое значение ползунка

### Дымовая сигнализация
Предоставляется коммунальными счетчиками.

- `nextLevel` (ro) идентификатор следующего уровня тревоги
    * 1 = Тихо
    * 2 = Acustic
    * 3 = Оптический
    * 4 = Внутренний
    * 5 = Внешний
    * 6 = Удаленный
- `nextLevelDelay` (ro) задержка следующего уровня в секундах
- `nextLevelDelayTotal` (ro) общая задержка следующего уровня в секундах
- `level` (ro) идентификатор текущего уровня тревоги
    * 1 = Тихо
    * 2 = Acustic
    * 3 = Оптический
    * 4 = Внутренний
    * 5 = Внешний
    * 6 = Удаленный
- `Датчики` (ро) список датчиков
- «ousticAlarm` (ro) состояние акустической тревоги ложно для неактивных и истинно для активных
- `testAlarm` (ro), активен ли testalarm
- `alarmCause` (ro) причина тревоги:
    * 1 = только детектор дыма
    * 2 = только вода
    * 3 = дым и вода
    * 4 = только температура
    * 5 = огонь и температура
    * 6 = температура и вода
    * 7 = огонь, температура и вода
- отметка времени `startTime` (ro) при запуске тревоги
- `timeServiceMode` (rw) задержка до отключения сервисного режима
- `mute` (wo) запись любого значения в это состояние отключает сирену
- `quit` (wo) запись любого значения в это состояние подтверждает дымовую сигнализацию

### Switch
Предоставляется виртуальными входными переключателями.

- `active` (rw) текущее состояние коммутатора

### TimedSwitch
Предусмотрен лестничный колодец и многофункциональные выключатели.

- `deactivationDelayTotal` (ro) секунд, как долго будет активен выход, если используется таймер
- `deactivationDelay` (ro) обратный отсчет, пока выход не будет деактивирован
    * 0 = выход отключен
    * -1 = выход постоянно включен
    * в противном случае он будет отсчитывать от deactivationDelayTotal
- `active` (wo) включает или отключает переключатель (без задержки деактивации)
- «пульс» (wo) пульсирует переключатель:
    * deactivationDelay = 0
        * Начнется обратный отсчет от deactivationDelayTotal до 0
    * если это переключатель лестничного колодца:
        * deactivationDelay = -1
            * Нет эффекта, будет постоянно включен.
        * deactivationDelay> 0
            * Возобновляет обратный отсчет
    * если это многофункциональный переключатель
        * выключает его (от обратного отсчета до постоянного состояния)

### Трекер
Предусмотрен лестничный колодец и многофункциональные выключатели.

- `records` (ro) список записей, возвращаемых с минисервера

### WindowMonitor
Предоставляется коммунальными счетчиками.

- `numOpen` (ro) количество открытых окон и дверей
- `numClosed` (ro) количество закрытых окон и дверей
- `numTilted` (ro) количество наклонных окон и дверей
- `numOffline` (ro) количество окон и дверей, которые не доступны
- `numLocked` (ro) количество заблокированных окон и дверей
- `numUnlocked` (ro) количество незапертых окон и дверей

Сумма значений из всех этих состояний равна количеству контролируемых окон и дверей. Окна / двери с двумя состояниями всегда будут учитываться в «худшем» состоянии.

Для каждого контролируемого окна / двери будет устройство с индексом в качестве идентификатора и заданного имени. У них есть следующие состояния:

- `закрыто` (ро) окно / дверь закрыта
- «наклонено» (ро) окно / дверь наклонены
- `открыть` (ро) окно / дверь открыта
- `заблокирован` (ро) окно / дверь закрыта
- `разблокировано` (ро) окно / дверь разблокирована

## Weather Server
Информация о погодном сервере предоставляется в виде устройства с несколькими каналами.
Устройство называется `WeatherServer`.
Это содержит:

- канал `Actual` с текущими значениями погоды
- один канал для каждого часа прогноза, называемый `HourXX`, где` XX` - это количество часов с этого момента

Каждый канал содержит следующие состояния:

- `barometricPressure`: числовое значение барометрического давления
- `barometricPressure-formatted`: форматированное значение барометрического давления с единицей измерения
- `dewPoint`: числовое значение точки росы
- `dewPoint-formatted`: форматированное значение точки росы с единицей измерения
- `воспринимаемая температура`: числовое значение воспринимаемой температуры
- `воспринимается температура-отформатировано`: форматируется воспринимаемое значение температуры с единицей измерения
- «осадки»: численное значение осадков
- `отформатированное количество осадков ': отформатированное значение количества осадков с единицей измерения
- `lativeHumidity`: числовое значение относительной влажности
- `lativeHumidity-formatted`: форматированное значение относительной влажности с единицей измерения
- `solarRadiation`: значение солнечного излучения
- `температура`: числовое значение температуры
- «температура в формате»: форматированное значение температуры с единицей измерения
- `timestamp`: отметка времени данных как` value.time` (время JavaScript)
- `weatherType`: числовое значение перечисления типа погоды
- `weatherType-text`: текстовое представление типа погоды
- `windDirection`: значение направления ветра
- `windSpeed`: значение скорости ветра
- `windSpeed-formatted`: отформатированное значение скорости ветра с единицей измерения
