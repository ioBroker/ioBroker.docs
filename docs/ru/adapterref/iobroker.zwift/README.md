# IoBroker.zwift
**Тесты:** ![Тестирование и выпуск](https://github.com/Flixhummel/ioBroker.zwift/workflows/Test%20and%20Release/badge.svg)

## Адаптер Zwift для ioBroker
Опрашивает API Zwift для получения данных о тренировках в режиме реального времени и делает их доступными, как указано в описании ioBroker. Отслеживайте свою мощность, частоту сердечных сокращений, частоту вращения педалей, скорость и многое другое в режиме реального времени во время езды на Zwift.

### Функции
— Данные о пассажирах обновляются в режиме реального времени каждые 5 секунд (настраивается).
- Расчет зон мощности в реальном времени (6-зонная модель Coggan, FTP автоматически считывается из вашего профиля Zwift)
- Подробные данные профиля Zwift: личность, статистика езды на велосипеде, статистика бега, майки, отрывы, серии побед и поражений.
- Индикатор состояния соединения (`info.connection`)
- Автоматическое обновление токена с резервным вариантом повторной аутентификации.
- Зашифрованное хранилище учетных данных
- Обновления метаданных применяются автоматически при перезапуске адаптера (исправления единиц измерения, добавление новых полей).

#### Зоны мощности
Состояние `powerZone` использует 6-зонную модель Коггана и рассчитывается автоматически на основе вашей текущей мощности и FTP. Значение FTP считывается из вашего профиля Zwift — ручная настройка не требуется.

| Зона | Название | % FTP |
|------|------|----------|
| 1 | Активное восстановление | < 55% |
| 2 | Выносливость | 55-75% |
| 3 | Темп | 76-90% |
| 4 | Порог лактата | 91-105% |
| 5 | VO2 Max | 106-120% |
| 6 | Анаэробная способность | > 120% |

Если в вашем профиле Zwift не указан FTP, состояние `powerZone` обновляться не будет.

### Как это работает
Адаптер аутентифицируется в API Zwift, используя ту же конечную точку, что и приложение Zwift Companion (`client_id=Zwift_Mobile_Link`). При запуске он получает ваш профиль Zwift (включая FTP), а затем опрашивает статус райдера через сервер ретрансляции игры, декодирует ответ protobuf, преобразует необработанные значения в удобочитаемые единицы измерения и обновляет дерево состояний ioBroker.

Если в вашем профиле задано значение FTP, адаптер вычисляет значение `powerZone` (1-6) в реальном времени при каждом цикле опроса, используя модель энергетических зон Коггана. Ручная настройка FTP не требуется.

Когда вы не катаетесь активно в Zwift, адаптер устанавливает `isRiding` в `false` и продолжает опрос без ошибок.

**Техническое примечание:** Объекты состояний создаются с использованием `extendObjectAsync`, а не `setObjectNotExistsAsync`. Это означает, что изменения метаданных (исправленные единицы измерения, переименованные состояния, новые поля) применяются автоматически при каждом перезапуске адаптера. Вам не нужно удалять и создавать объекты заново после обновления.

### Идеи для умного дома
Как заявляет ioBroker, благодаря доступу к данным Zwift вы можете создавать автоматизации, которые позволят вам тренироваться в помещении по всему дому.

**Иммерсивное освещение**

— Меняйте цвет светодиодных лент или ламп Hue в зависимости от зон сердечного ритма: синий — для восстановления, зеленый — для выносливости, желтый — для темпа, красный — для пороговой нагрузки, мигающий красный — для VO2max.
— Используйте состояние `powerZone` (1-6) для непосредственного управления цветовыми схемами — для самостоятельного расчета зон не требуется никаких скриптов.
- Изменение цвета подсветки в зависимости от мощности — чем сильнее вы нажимаете, тем интенсивнее свечение.
— Имитация высоты с помощью яркости освещения: приподнимайтесь при наборе высоты и увеличивайте яркость при снижении.
- Включите свет в комнате, когда получите Ride On.

**Адаптивное аудио**

— Автоматическое переключение плейлистов в зависимости от вашей зоны частоты сердечных сокращений или мощности — расслабляющая музыка для разминки, высокий темп для интервальных тренировок.
- Воспроизводится звуковой эффект при пересечении круга или достижении определенного количества сожженных калорий.
- Регулярно озвучивайте свои текущие статистические данные с помощью функции преобразования текста в речь.

**Приборные панели и дисплеи**

— Отображение мощности, частоты сердечных сокращений, скорости и частоты вращения педалей в режиме реального времени на настенном планшете или смарт-дисплее.
— Отображение пройденного расстояния, набора высоты и количества сожженных калорий на информационной панели в вашей «зоне боли».
— Создайте панель мониторинга VIS с вашей общей статистикой за все время, полученной из данных профиля — общее количество километров, общий набор высоты, уровень XP.

**Кондиционер**

- Включайте «умный» вентилятор, когда частота сердечных сокращений превышает пороговое значение, и выключайте его во время перерывов между ударами сердца.
- Увеличивайте скорость вращения вентилятора пропорционально потребляемой мощности.
- Активируйте кондиционер, когда количество сожженных калорий превысит определенное значение.

**Самодельное оборудование**

— Используйте данные о высоте/градиенте для управления самодельным механизмом качания или наклона тренажера — создайте свой собственный симулятор подъема в гору, похожий на Wahoo KICKR CLIMB.
— Управляйте сервоприводом или линейным актуатором через ioBroker, чтобы наклонять раму велосипеда в реальном времени в зависимости от изменения градиента в игре.

**Мотивация и геймификация**

- Запустите машину для конфетти или праздничную подсветку, когда завершите поездку или установите личный рекорд.
— Отправляйте себе уведомление в Telegram или Pushover с кратким описанием поездки, когда значение `isRiding` изменится на `false`.
— Отслеживайте пройденное расстояние за неделю на семисегментном дисплее или экране с электронными чернилами в коридоре.
- Загорится индикатор выполнения (светодиодная лента), показывающий процент завершения маршрута.

**Семья и домохозяйство**

- Включайте индикатор "Не беспокоить" снаружи комнаты всякий раз, когда значение параметра `isRiding` равно `true`.
- Автоматическое отключение звука дверного звонка во время сессии Zwift
- Отправьте сообщение на умную колонку вашей семьи: «Папа катается в Zwift, ориентировочное время финиша: X минут»
