# IoBroker.open-meteo-weather
![Тестирование и выпуск](https://github.com/H5N1v2/iobroker.open-meteo-weather/workflows/Test%20and%20Release/badge.svg)

## Важная информация:
В этот адаптер теперь интегрированы Open Meteo Weather и Open Meteo PV Forecast. Поддержка Open Meteo PV Forecast прекращена.

---

**Этот адаптер использует библиотеки Sentry для автоматического сообщения разработчикам об исключениях и ошибках в коде.** Для получения более подробной информации и инструкций по отключению сообщений об ошибках, пожалуйста, обратитесь к разделу [Документация по плагину Sentry](https://github.com/ioBroker/plugin-sentry#plugin-sentry)! Использование системы отчетности Sentry начинается с версии js-controller 3.0.

Я использую собственный сервер Sentry на базе Glitchtip.

**Адаптер сервиса прогнозирования погоды и солнечной энергии Open-Meteo для ioBroker.**

Этот адаптер предоставляет точные данные о погоде, прогнозы, информацию о качестве воздуха, содержании пыльцы и прогнозы выработки фотоэлектрической энергии на основе [Open-Meteo.com](https://open-meteo.com/). Он бесплатен для некоммерческого использования (менее 10 000 вызовов API в день) и не требует регистрации ключа API, что делает процесс настройки чрезвычайно простым.

---

# Особенности Погода
* **Текущие данные о погоде:** Получение в режиме реального времени данных о температуре, влажности, атмосферном давлении и ветре.
* **Гибкие прогнозы:** Настраиваемое количество дней прогноза и почасовое разрешение.
* **Качество воздуха и пыльца:** Дополнительные данные о твердых частицах (PM2.5, PM10), а также о различных типах пыльцы (ольха, береза, трава и т. д.).
* **Автоматическая очистка:** Адаптер автоматически очищает структуру объекта, если периоды прогнозирования сокращаются или изменяются в конфигурации.
* **Многоязычная поддержка:** Поддерживает 11 языков (включая английский, немецкий, польский, русский, французский, китайский и др.).
* **Система единиц измерения:** Плавное переключение между метрической (°C, км/ч) и имперской (°F, миль/ч) системами.
* **Несколько местоположений:** Добавляйте несколько местоположений.
* **Ночные значки:** Вы можете выбрать один из двух наборов ночных значков: «Светлый» и «Темный». Это упростит подбор значка под фон вашего помещения.

### Значки направления ветра
В настройках адаптера можно выбрать один из двух стилей отображения направления ветра:

* Направление ветра (куда дует ветер): Это стандартная настройка. Стрелка указывает направление ветра. (Например: при северном ветре стрелка указывает на юг).

* Источник ветра (откуда дует ветер): В этом стиле используются значки из подпапки direct_2. Стрелка указывает на источник ветра. (Пример: северный ветер обозначается стрелкой, указывающей на север, или конкретным значком «источника»).

|Настройки | Путь к значку | Поведение |
|:---|:---|:---|
|Направление ветра (куда дует ветер) | /icons/wind_direction_icons/*.png | Указывает на пункт назначения |
|Источник ветра (место, откуда дует ветер) |/icons/wind_direction_icons/direct_2/*.png | Указывает на источник ветра |

### Данные о качестве воздуха
Адаптер предоставляет актуальные данные о качестве воздуха и ежедневный прогноз на ближайшие дни (с возможностью настройки на 1, 3 или 6 дней).

Эффективная обработка данных: В то время как API Open-Meteo предоставляет только необработанные почасовые данные, этот адаптер интеллектуально агрегирует эти значения. Он автоматически рассчитывает суточные максимумы для всех загрязняющих веществ и уровней пыльцы. Это позволяет получить наиболее актуальные данные (пиковые уровни воздействия) без перегрузки базы данных сотнями почасовых точек данных.

Функции:

* Ежедневные пиковые значения: Получите максимально ожидаемые значения для PM2.5, PM10, озона и различных типов пыльцы.

* Удобочитаемый формат: Уровни пыльцы автоматически сопоставляются с описательными категориями (например, «Отсутствует», «Низкий», «Умеренный», «Высокий»).

* Интеллектуальная очистка: объекты для прогнозируемых дней автоматически создаются или удаляются в зависимости от ваших настроек, чтобы поддерживать чистоту дерева объектов.

## Конфигурация
После установки настройте следующие поля в параметрах экземпляра:

1. **Местоположение:** Укажите ваше местоположение или желаемое название.
2. **Координаты (широта и долгота):** Добавьте свои координаты. Вы можете найти их, нажав кнопку «Найти координаты с помощью OpenStreetMap», или оставить поля пустыми, чтобы использовать системные координаты.
3. **Часовой пояс:** Установите часовой пояс в выпадающем меню. По умолчанию установлено значение «Авто», которое автоматически подстраивается под ваши координаты.
4. **Интервал обновления:** Временной интервал в минутах (по умолчанию: 30 мин).
5. **Прогнозируемые дни:** Количество дней для ежедневного обзора (0–16 дней).
6. **Почасовой прогноз:** Включите или отключите эту опцию и укажите количество часов (например, следующие 24 часа). Например, hour0 — это текущий час, hour1 — следующий час и так далее.
7. **Дополнительные данные:** Флажки для указания данных о пыльце и качестве воздуха.
8. **Единицы измерения:** Выберите между метрической и имперской системами.

## Виджет погоды
Этот адаптер предлагает два способа отображения данных о погоде в вашей визуализации:

### 1. Встроенный виджет (стандартный)
Начиная с версии **3.1.0**, адаптер может автоматически генерировать предварительно настроенный HTML-виджет для каждого местоположения.

**Способ применения:**

1. **Включить:** Установите флажок «Создать виджет» в настройках экземпляра для вашего местоположения.
2. **Найти состояние:** Адаптер создаст состояние с именем `htmlWidget` (в каталоге `open-meteo-weather.0.yourLocation.htmlWidget`).
3. **В VIS/VIS2:** * Перетащите стандартный **виджет HTML** на ваше представление.
* Установите свойство "HTML" этого виджета в соответствии с привязкой вашего состояния: `{open-meteo-weather.0.yourLocation.htmlWidget}`.
* Отрегулируйте ширину и высоту контейнера виджета в соответствии с содержимым.

**Настройка:** Основные параметры, такие как размер шрифта, время и дни прогноза погоды, можно изменять только в конфигурации адаптера.

**Идеально подходит для:** Пользователей, которым нужен быстрый, красивый и не требующий обслуживания дисплей.

**Примечание:** Для оптимального отображения в браузерах на настольных компьютерах не используйте дополнительные CSS-границы или тени в настройках редактора VIS. Виджет поставляется со своим собственным оптимизированным стилем.

### 2. Расширенный скрипт виджета (полная настройка)
Если вы хотите внести существенные изменения в дизайн, добавить собственный CSS-код или расширить логику:

* **Ссылка:** Используйте [VIS2-widget-script-om-weather](https://github.com/H5N1v2/VIS2-widget-script-om-weather).
* **Наилучшим образом подходит для:** Опытных пользователей, желающих получить полный контроль над каждым HTML-тегом и свойством CSS.

## Иконки и визуализация
Адаптер предоставляет динамические пути к значкам, которые можно использовать непосредственно в визуализациях, таких как **vis, iQontrol или Jarvis**.

* **Значки погоды:** Находятся по адресу `weather.current.icon_url`. Адаптер автоматически различает день и ночь (например, Солнце и Луна).
* **Направление ветра:** Статические траектории под значком `wind_direction_icon` отображают стрелку компаса, соответствующую значению в градусах (направление отображения можно выбрать).
* **Предупреждение о порывах ветра:** При скорости ветра выше примерно 39 км/ч (6 баллов по шкале Бофорта) под иконкой `wind_gust_icon` отображается значок предупреждения, обозначающий уровни 0–4.
* **Значки фаз Луны:** Значки Луны можно найти в разделе `moon_phase_icon`, они отображают фазы Луны.
* **Несколько значков:** Вы можете выбрать между статическими и анимированными значками (от [basmilius](https://github.com/basmilius/meteocons)).

# Функция прогнозирования солнечной активности (если включена)
* **Поддержка нескольких местоположений:** Поддержка нескольких фотоэлектрических систем/мест, например, для восточной/западной ориентации.
* **Почасовой прогноз:** Подробный прогноз выходной мощности, температуры, облачности и продолжительности солнечного сияния.
* **Ежедневный прогноз:** Сводка ожидаемого потребления энергии (Вт·ч) на срок до 14 дней.
* **15-минутный прогноз:** 15-минутный прогноз на текущий день и 24 часа.
* **Физическое моделирование:**
* **Наклон и азимут:** Расчет интенсивности излучения на основе ориентации панели.
* **Температура фотоэлектрического модуля:** Оценка температуры ячейки с учетом температуры окружающей среды, интенсивности излучения и скорости ветра (модель Файмана).
* **Продолжительность солнечного сияния:** Перевод продолжительности солнечного сияния в минуты в час.
* **Агрегация:** Автоматическое суммирование данных по всем местоположениям (общий прогноз) за сутки, час и 15 минут.
* **Системная интеграция:** Автоматическое получение координат местоположения из конфигурации системы ioBroker, если они не заданы вручную.
* **Температура фотоэлектрического модуля:** Расчетная температура фотоэлектрического модуля, рассчитанная на основе модели Файмана.

## Конфигурация
### Основные настройки
* **Прогнозируемые часы:** Период времени для почасового просмотра (от 3 до 48 часов).
* **Прогнозируемые дни:** Продолжительность ежедневного прогноза (от 3 до 14 дней).
* **Интервал обновления:** Частота обновления данных (15, 30, 60 минут или один раз перед восходом солнца).

### Настройки точек данных
* **Скользящий или фиксированный график:** Если выбран вариант «Скользящий график», почасовой прогноз всегда будет показывать следующие часы, начиная с текущего. Если выбран вариант «Фиксированный график», почасовой прогноз будет показывать фиксированные временные интервалы (например, 00:00-23:00) независимо от текущего времени.
* **Прогноз на 15 минут:** Если эта функция включена, будут созданы дополнительные штаты для 15-минутного прогноза (до 24 часов на текущий день). Обратите внимание, что доступность 15-минутных данных зависит от API Open-Meteo и может варьироваться в зависимости от местоположения или времени.

### Места расположения (таблица)
Для каждого местоположения необходимо задать следующие значения:

1. **Имя:** Уникальный идентификатор (прошедший очистку по идентификатору объекта).
2. **Широта/Долгота:** GPS-координаты (необязательно: в противном случае используются системные значения).
3. **Наклон:** Угол наклона модулей (0° = горизонтальное положение, 90° = вертикальное положение).
4. **Азимут:** Ориентация (-180° до 180°, 0° = Юг, -90° = Восток, 90° = Запад).
5. **Мощность (кВтп):** Установленная пиковая мощность системы.
6. **Часовой пояс:** Выбор местного часового пояса (по умолчанию: Авто).

![Логотип](../../../en/adapterref/iobroker.open-meteo-weather/admin/img/doc.png)

### Глобальные параметры, настраиваются только при наличии нескольких местоположений!
* **Общая сумма (ежедневно):** Создает канал `sum_peak_locations_Daily`, суммирующий урожайность всех систем.
* **Общая сумма (за час):** Создает канал `sum_peak_locations_Hourly` для суммарной почасовой производительности.
* **Общая сумма (за 15 минут):** Создает канал `sum_peak_locations_15_Minutely` для общего прогноза на 15 минут.

## Технические характеристики и расчеты
### Модель температуры фотоэлектрической системы
Адаптер использует **модель Файмана** для оценки температуры модуля. Эта модель учитывает ветровое охлаждение, которое напрямую влияет на эффективность: `pvTemp = Ambient Temperature + Irradiance / (25 + 6.84 * Wind Speed)`.
