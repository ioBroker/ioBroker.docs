# IoBroker.open-meteo-weather
![Тестирование и выпуск](https://github.com/H5N1v2/iobroker.open-meteo-weather/workflows/Test%20and%20Release/badge.svg)

**Этот адаптер использует библиотеки Sentry для автоматического сообщения разработчикам об исключениях и ошибках в коде.** Для получения более подробной информации и инструкций по отключению сообщений об ошибках, пожалуйста, обратитесь к разделу [Документация по плагину Sentry](https://github.com/ioBroker/plugin-sentry#plugin-sentry)! Использование системы отчетов Sentry начинается с версии js-controller 3.0.

Я использую собственный сервер Sentry на базе Glitchtip.

**Адаптер Open-Meteo Weather Service для ioBroker.**

## Во-первых: если вам нужен виджет специально для этого адаптера, создайте его, используя [VIS2-widget-script-om-weather](https://github.com/H5N1v2/VIS2-widget-script-om-weather).
Этот адаптер предоставляет точные данные о погоде, прогнозы, информацию о качестве воздуха и содержании пыльцы, работая на основе [Open-Meteo.com](https://open-meteo.com/). Он бесплатен для некоммерческого использования (менее 10 000 вызовов API в день) и не требует регистрации ключа API, что делает процесс настройки чрезвычайно простым.

## Функции
* **Текущие данные о погоде:** Получение в режиме реального времени данных о температуре, влажности, атмосферном давлении и ветре.
* **Гибкие прогнозы:** Настраиваемое количество дней прогноза и почасовое разрешение.
* **Качество воздуха и пыльца:** Дополнительные данные о твердых частицах (PM2.5, PM10), а также о различных типах пыльцы (ольха, береза, трава и т. д.).
* **Автоматическая очистка:** Адаптер автоматически очищает структуру объекта, если периоды прогнозирования сокращаются или изменяются в конфигурации.
* **Многоязычная поддержка:** Поддерживает 11 языков (включая английский, немецкий, польский, русский, французский, китайский и др.).
* **Система единиц измерения:** Плавное переключение между метрической (°C, км/ч) и имперской (°F, миль/ч) системами.
* **Несколько местоположений:** Добавляйте несколько местоположений.
* **Ночные значки:** Вы можете выбрать один из двух наборов ночных значков: «Светлый» и «Темный». Это упростит подбор значка под фон вашего помещения.

### Значки направления ветра
В настройках адаптера можно выбрать один из двух стилей отображения направления ветра:

* Направление ветра (куда дует ветер): Это стандартная настройка. Стрелка указывает направление ветра. (Например: при северном ветре стрелка указывает на юг).

* Источник ветра (откуда дует ветер): В этом стиле используются значки из подпапки direct_2. Стрелка указывает на источник ветра. (Пример: северный ветер обозначается стрелкой, указывающей на север, или конкретным значком «источника»).

|Настройки | Путь к значку | Поведение |
|:---|:---|:---|
|Направление ветра (куда дует ветер) | /icons/wind_direction_icons/*.png | Указывает на пункт назначения |
|Источник ветра (место, откуда дует ветер) |/icons/wind_direction_icons/direct_2/*.png | Указывает на источник ветра |

### Данные о качестве воздуха
Адаптер предоставляет актуальные данные о качестве воздуха и ежедневный прогноз на ближайшие дни (с возможностью настройки на 1, 3 или 6 дней).

Эффективная обработка данных: В то время как API Open-Meteo предоставляет только необработанные почасовые данные, этот адаптер интеллектуально агрегирует эти значения. Он автоматически рассчитывает суточные максимумы для всех загрязняющих веществ и уровней пыльцы. Это позволяет получить наиболее актуальные данные (пиковые уровни воздействия) без перегрузки базы данных сотнями почасовых точек данных.

Функции:

* Ежедневные пиковые значения: Получите максимально ожидаемые значения для PM2.5, PM10, озона и различных типов пыльцы.

* Удобочитаемый формат: Уровни пыльцы автоматически сопоставляются с описательными категориями (например, «Отсутствует», «Низкий», «Умеренный», «Высокий»).

* Интеллектуальная очистка: объекты для прогнозируемых дней автоматически создаются или удаляются в зависимости от ваших настроек, чтобы поддерживать чистоту дерева объектов.

## Конфигурация
После установки настройте следующие поля в параметрах экземпляра:

1. **Местоположение:** Укажите ваше местоположение или желаемое название.
2. **Координаты (широта и долгота):** Добавьте свои координаты. Вы можете найти их, нажав кнопку «Найти координаты с помощью OpenStreetMap», или оставить поля пустыми, чтобы использовать системные координаты.
3. **Часовой пояс:** Установите часовой пояс в выпадающем меню. По умолчанию установлено значение «Авто», которое автоматически подстраивается под ваши координаты.
4. **Интервал обновления:** Временной интервал в минутах (по умолчанию: 30 мин).
5. **Прогнозируемые дни:** Количество дней для ежедневного обзора (0–16 дней).
6. **Почасовой прогноз:** Включите или отключите эту опцию и укажите количество часов (например, следующие 24 часа). Например, hour0 — это текущий час, hour1 — следующий час и так далее.
7. **Дополнительные данные:** Флажки для указания данных о пыльце и качестве воздуха.
8. **Единицы измерения:** Выберите между метрической и имперской системами.

## Иконки и визуализация
Адаптер предоставляет динамические пути к значкам, которые можно использовать непосредственно в визуализациях, таких как **vis, iQontrol или Jarvis**.

* **Значки погоды:** Находятся по адресу `weather.current.icon_url`. Адаптер автоматически различает день и ночь (например, Солнце и Луна).
* **Направление ветра:** Статические траектории в разделе `wind_direction_icon` отображают стрелку компаса, соответствующую значению в градусах.
* **Предупреждение о порывах ветра:** При скорости ветра выше примерно 39 км/ч (6 баллов по шкале Бофорта) под иконкой `wind_gust_icon` отображается значок предупреждения, обозначающий уровни 0–4.
* **Значки фаз Луны:** Значки Луны можно найти в разделе `moon_phase_icon`, они отображают фазы Луны.

## Данные (фрагмент)
| Папка | Описание |
|:---|:---|
| `air.current` | Качество воздуха и уровень пыльцы в текстовом и числовом виде |
| `weather.current` | Текущие измерения (температура, точка росы, ветер и т. д.) |
| `weather.forecast.dayX` | Ежедневный прогноз на день X |
| `weather.forecast.hourly.hourX` | Почасовые данные за полный час |
| `info.lastUpdate` | Отображает дату и время последнего обновления |
| `info.lastUpdate` | Отображает дату и время последнего обновления |
