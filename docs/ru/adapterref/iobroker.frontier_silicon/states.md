# Документация по штатам
Пока адаптер считывает настройки устройства, создаются объекты ioBroker и состояния.
Состояния могут быть только для чтения (`ro`) или для чтения и записи (`rw`) *хорошо, для кнопок также возможна запись*.

- аудио

Основные настройки звука. Регулировка эквалайзера пока не реализована.

- maxVolume (`number, ro`)

Максимально выбираемый уровень громкости

- mute (`boolean, rw`)

`true` если устройство отключено, `false` в противном случае

- объем (`number, rw`)
  - контроль
- уменьшение и увеличение громкости

Вводит/уменьшает объем на 1

- устройство

- friendlyName (`string, rw`)
- power (`boolean, rw`)
- radioId (`string, ro`)

Предполагаю, что это MAC-адрес устройства.

- версия (`string, ro`)

Версия программного обеспечения

- webfsapi (`string, ro`)

Адрес API

- информация

- connection (`boolean, ro`)

Индикатор подключения адаптера

- СМИ

- состояние (`string, ro`)

Допустимые значения:

- 0: «БЕЗДЕЙСТВИЕ»
- 1: «БУФЕРИЗАЦИЯ»
- 2: «ИГРА»
- 3: «ПАУЗА»
- 4: «ОТКАТКА»
- 5: «ОШИБКА»
- 6: «ОСТАНОВЛЕНО»
- 7: “ERROR_POPUP”

- control (`boolean, wo`)

- 0: «СТОП»
- 1: «ИГРАТЬ»
- 2: «ПАУЗА»
- 3: «ДАЛЕЕ»
- 4: «ПРЕДЫДУЩИЙ»

Не стоит воспринимать следующие названия слишком серьёзно. Радиостанция использует их по-разному в разных режимах.

- альбом (`string, ro`)
- artist (`string, ro`)
- graphic (`string, ro`)

Используйте этот URL-адрес, чтобы получить обложку альбома или логотип радиостанции.

- имя (`string, ro`)
- строка (`string, ro`)
- title (`string, ro`)

- режимы

- currentNavIndex (`number, ro`)

Указывает индекс в текущем списке навигации.
Полный список навигации считывается порциями по 25 элементов.
Собственная часть этого объекта также содержит некоторые другие свойства текущей навигации, такие как все имена навигации, количество элементов в текущем полном списке навигации, а также тип и подтип текущих элементов, на которые указывает индекс.
Типы навигации: "0": "Каталог", "1": "Воспроизводимый элемент", "2": "Каталог поиска", "3": "Неизвестно", "4": "Элемент формы", "5": "Элемент сообщения", "6": "Вход в Amazon" Подтипы навигации: "0": "Нет", "1": "Станция", "2": "Подкаст", "3": "Трек", "4": "Текст", "5": "Пароль", "6": "Параметры", "7": "Отправить", "8": "Кнопка", "9": "Отключено"

- currentNavKey (`number, rw`)

Указывает или задает выбранную клавишу для всего списка навигации.

- currentNavList (`string, ro`)

Строка в формате JSON, содержащая текущий выбранный фрагмент списка навигации.
Это массив ключ/значение, содержащий все свойства каждого элемента, например:

{"key":"116","name":"1LIVE diGGi ","type":"1","subtype":"1","graphicuri":"","artist":"","contextmenu":"0"} Его можно использовать в скрипте JavaScript для динамической генерации объекта JSON или HTML-страницы для использования в виджете медиаплеера.

- currentNavName (`string, ro`)

Название текущего выбранного элемента в списке навигации, например, каталог или воспроизводимый элемент.

- navigationBack (`boolean, wo`)

вернуться на один уровень назад в дереве навигации

- navigationDown (`boolean, wo`)

перемещайтесь вниз внутри текущего уровня дерева навигации.

- navigationUp (`boolean, wo`)

перемещаться вверх внутри текущего уровня дерева навигации

- navigationSelect (`boolean, wo`)

Выбирает текущий индексированный элемент. Если выбранный элемент является каталогом, открывается следующий уровень в дереве навигации. Если выбранный элемент является воспроизводимым элементом, воспроизведение начнется автоматически.

- navigationHome (`boolean, wo`)

вернуться на верхний уровень дерева навигации

- navigationSearch (`string, rw`)

Позволяет ввести поисковый запрос, который будет отправлен на устройство. Однако устройство начнет поиск только в том случае, если нажата кнопка navigationSelect и выбранный элемент имеет тип SearchDirectory. Поисковый запрос следует ввести непосредственно перед нажатием кнопки navigationSelect.

- presetDown (`boolean, wo`)

Прокрутите вниз в пределах предустановленных режимов. Следующая станция или трек начнут воспроизводиться автоматически.

- presetUp (`boolean, wo`)

Перемещайтесь вверх в пределах предустановленных режимов. Следующая станция или трек автоматически начнут воспроизводиться.

- readPresets (`boolean, wo`)

Перечитывает все предустановки

- selectPreset (`number, rw`)

Используется для получения или выбора предустановки.
Имейте в виду, что адаптер делает предположение, поскольку это значение нельзя прочитать из API.

- выбрано (`number, rw`)

Указывает или выбирает выбранный режим. Также может быть выбран с помощью `modes.{number}.switchTo`

- selectedLabel (`string, ro`)

Указывает на обозначение выбранного режима.

- `{number}`

- id (`string, ro`)

Название этого режима

- key (`number, ro`)

Индекс этого режима. Равен `mode.{number}` из дерева объектов и может быть записан в `modes.selected`.

- выбираемый (`логическое значение, ro`)

`true` если этот режим можно выбрать вручную.

- streamable (`boolean, ro`)

Присутствует только на устройствах с поддержкой многокомнатного режима. `true` если этот режим может использоваться в качестве источника для нескольких многокомнатных устройств.

- switchTo (`boolean, wo`)

Выбирает этот режим.

- предустановки

- available (`boolean, ro`)

Указывает, доступны ли предустановки для этого режима.

- `{number}`

Индекс этого предустановленного значения. Равен `mode.*.presets.{number}.key`.

- key (`number, ro`)

Индекс этого предустановленного значения. Равен `mode.*.presets.{number}` из дерева объектов и может быть записан в `modes.selectPreset`.

- имя (`string, ro`)

Название этого пресета

- отзыв (`логическое значение, wo`)

Выбирает этот предустановленный режим **и** соответствующий режим.

- set (`boolean, wo`)

Устанавливает предустановку на текущую воспроизводимую радиостанцию или трек. Если вы попытаетесь установить предустановку режима X, когда воспроизводится радиостанция другого режима, вы получите предупреждение.
