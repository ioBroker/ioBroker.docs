# IoBroker.dreame
**Тесты:** ![Тестирование и выпуск](https://github.com/TA2k/ioBroker.dreame/workflows/Test%20and%20Release/badge.svg)

**Этот адаптер использует библиотеки Sentry для автоматического сообщения разработчикам об исключениях и ошибках в коде.** Для получения более подробной информации и сведений о том, как отключить отправку сообщений об ошибках, см. [Документация по плагину Sentry](https://github.com/ioBroker/plugin-sentry#plugin-sentry)! Отправка сообщений Sentry используется начиная с js-controller 3.0.

## Адаптер Dreame для ioBroker
Адаптер для роботов-пылесосов и роботов-газонокосилок Dreame и MOVA.

**Поддерживаемые бренды:** Dreame, MOVA (выберите в настройках адаптера)

**Протестировано с:** L10, L20, X40, A2 1200 (газонокосилка), MOVA 600, MOVA 1000

---

## Конфигурация
| Настройки | Описание |
| --- | --- |
| Облачный сервис | Выберите **Dreame** или **MOVA** в зависимости от вашего приложения |
| Электронная почта приложения | Ваш адрес электронной почты для входа в приложение Dreame/MOVA |
| Пароль приложения | Ваш пароль от приложения Dreame/MOVA |
| Получить карту | Включить отрисовку карты (более высокая загрузка ЦП) |
| Интервал обновления | Интервал опроса в минутах |

Устройства MOVA (600, 1000) используют ту же облачную инфраструктуру, что и Dreame, но с другими доменами. Выберите **MOVA**, если вы используете приложение MOVA.

#### Уборка номера
`dreame.0.XXXX.remote.start-custom-clean`

```json
[
  { "piid": 1, "value": 18 },
  { "piid": 10, "value": "{\"selects\":[[X,1,3,2,1]]}" }
]
```

X = ID комнаты. Несколько комнат: `{\"selects\":[[X,1,3,2,1],[Y,1,3,2,1]]}`

#### Переключить карту
`dreame.0.XXXXXXX.remote.customCommand`:

```json
{ "siid": 6, "aiid": 2, "in": [{ "piid": 4, "value": "{\"sm\":{},\"mapid\":X}" }] }
```

X = mapId (см. `dreame.0.XXXX.status.map-list`)

### Схема газонокосилки
Данные карты получаются через API Dreame iotuserdata (а не через MQTT, как у пылесосов).

| Штат | Описание |
| --------------- | ---------------------------------------- |
| mapImage | Карта, отрисованная в формате PNG (URL данных base64) |
| slot0.zone_X | Данные зоны (название, площадь, время кошения) |
| Путь скашивания | Исходные координаты пути скашивания |
| Настройки | Настройки кошения по зонам |
| расписание | График покоса травы |
| 3dmap-url | URL для загрузки 3D-карты LIDAR (предварительно подписана) |
| Прогресс создания 3D-карты | Прогресс генерации 3D-карты (0-100%) |

**Опрос карты:** Карта загружается при запуске адаптера и с помощью кнопки `fetchMap`. Во время активного кошения (статусы 1, 3, 5, 11) карта автоматически опрашивается каждые 30 секунд для отслеживания траектории кошения.

**Отображение карты:** Требуется дополнительный пакет npm `canvas`. На карте отображаются зоны (зеленый цвет), контуры (белые линии), траектория скашивания (желтый цвет), запрещенные зоны (красный цвет) и препятствия (красные круги).

**3D-карта LIDAR:** Нажмите `generate-3dmap`, чтобы запустить сканирование и загрузку 3D-карты облака точек. Загруженный файл представляет собой файл PCD (Point Cloud Data), который можно просмотреть с помощью таких инструментов, как CloudCompare или MeshLab. Ход выполнения отслеживается в `3dmap-progress`. После завершения предварительно подписанный URL-адрес для загрузки записывается в `3dmap-url`. URL-адрес является временным и истекает через несколько часов.

#### Пользовательские команды для газонокосилки
Через `dreame.0.XXXXXX.remote.customCommand`:

```json
{
  "siid": 5,
  "aiid": 9,
  "in": [{ "order": 4, "region": [1], "type": "order" }]
}
```
