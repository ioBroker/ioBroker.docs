# Устранение неполадок ioBroker — подробное руководство
## Введение для начинающих
ioBroker — это мощная платформа для умного дома на базе Node.js, которая соединяет различные IoT-устройства и сервисы через адаптеры. Для начинающих важно понимать, что стабильная установка ioBroker основана на пяти основных принципах, которые следует систематически проверять при возникновении проблем.

## 5 основных принципов стабильной установки ioBroker
### 1. **Выбор и настройка операционной системы**
**Рекомендуемые операционные системы:**

- **Debian** (стабильная версия): Максимальная стабильность, самые длительные циклы поддержки, идеально подходит для производственных систем.
- **Ubuntu LTS**: Хороший компромисс между актуальностью и стабильностью, большое сообщество.
- **Raspberry Pi OS**: Оптимизирована для оборудования Pi, основана на Debian.

- **Возможно, но не рекомендуется:**

- **macOS**

**Основные системные требования:**

- Не менее 2 ГБ оперативной памяти (для больших систем рекомендуется 4 ГБ)
- Не менее 16 ГБ места для хранения данных (32 ГБ и более для крупных установок)
- Стабильное интернет-соединение для обновлений и загрузки адаптера.
- Доступ по SSH для проведения технического обслуживания.

**Почему выбор ОС важен:** Неправильный выбор операционной системы приводит к повторяющимся проблемам. Промежуточные версии Ubuntu имеют короткий цикл поддержки и быстро устаревают. Установка Windows возможна, но не рекомендуется.

### 2. **Техническое обслуживание операционной системы**
**Регулярный план технического обслуживания:**

```bash
# Wöchentlich ausführen:
sudo apt update          # Paketlisten aktualisieren
sudo apt upgrade -y      # Sicherheitsupdates installieren
sudo apt autoremove      # Nicht benötigte Pakete entfernen
```

```bash
# Monatliche Vollwartung:
sudo apt update
sudo apt full-upgrade    # Größere Systemupdates
sudo apt autoclean       # Paket-Cache bereinigen
sudo reboot             # Neustart nach Kernel-Updates
```

**Автоматизация обслуживания:** Опытные пользователи могут использовать скрипты обслуживания, которые автоматически выполняют обновления системы и ioBroker. Однако их следует использовать только после тщательного тестирования в производственной среде.

### 3. **Правильная установка ioBroker**
**Рекомендуется однолинейная установка:**

```bash
curl -sLf https://iobroker.net/install.sh | bash -
```

**Что делает скрипт установки:**

- Автоматически создает пользователя `iobroker`
- Устанавливает правильную версию Node.js
- Настройте все необходимые каталоги и права доступа.
- Устанавливает базовый адаптер (администратор, обнаружение).
- Настраивает службы автозапуска

**Избегайте распространенных ошибок при установке:**

- **Не** устанавливайте от имени root.
- **Не** устанавливайте Node.js вручную (скрипт делает это автоматически).
- **Не** используйте команду sudo перед командой curl.
- Всегда используйте свежую, обновленную систему.

### 4. **Работа с нужным пользователем**
**Понимание концепции пользователя:** ioBroker работает под собственной системной учетной записью пользователя `iobroker`, **а не** под учетной записью root. Это критически важный аспект безопасности.

**Иерархия пользователей:**

- **Ваш пользователь для входа** (например, pi, ubuntu): Для входа по SSH и администрирования системы.
- **iobroker-User**: Выполняет все процессы ioBroker, имеет ограниченные права sudo.
- **root**: Только для администрирования системы, никогда не для работы с ioBroker.

**Решение типичных юридических проблем:**

```bash
# Wenn ioBroker-Befehle nicht funktionieren:
sudo usermod -aG iobroker $(whoami)  # Ihren User zur iobroker-Gruppe hinzufügen
# Danach: Aus- und wieder einloggen

# Fixer-Script bei Rechteproblemen:
iob fix
# als Fallback, falls die Kurzform nicht greift
curl -sL https://iobroker.net/fix.sh | bash -
```

**Чего НИКОГДА НЕ СЛЕДУЕТ ДЕЛАТЬ:**

- Запустите ioBroker с помощью команды `sudo iobroker ...`
- Работает от имени пользователя root для обычных операций ioBroker.
- Вручную "исправить" права доступа к файлу с помощью команды chmod 777.

### 5. **Поддерживайте Node.js в актуальном состоянии**
**Понимание совместимости:**

- **js-controller 7.x**: Node.js 18.x, 20.x, 22.x, 24.x
- **js-controller 6.x**: Node.js 18.x, 20.x, 22.x
- **js-controller 5.x**: Node.js 16.x, 18.x, 20.x
- **Устаревшие версии**: поддержка версий Node.js ниже 20.x прекращена.

**Обновите Node.js корректно:**

```bash
# Moderne Methode (seit 2024):
iob stop                 # ioBroker stoppen
iob fix                  # System reparieren
iob nodejs-update     # Auf empfohlene Node.js 20.x wechseln
sudo reboot             # System neu starten
iob start               # ioBroker starten
```

**Почему обновления Node.js крайне важны:**

— Обновления безопасности устраняют уязвимости
— Для новых адаптеров требуются современные версии Node.js.
- Улучшения производительности
- LTS-версии (20, 22, 24) стабильны и поддерживаются в течение длительного времени.

**Частота обновления:**

- **Обновления безопасности**: Установите немедленно
- **Незначительные обновления**: Проверяйте ежемесячно
- **Крупные обновления**: основаны на отзывах сообщества, обновления выйдут не сразу после релиза.

## Дополнительные важные понятия для начинающих
### Настройка и использование SSH-доступа
SSH (Secure Shell) — это стандартный способ управления системами ioBroker:

- **Windows**: Используйте PuTTY или Windows Terminal.
- **macOS/Linux**: Используйте встроенное приложение «Терминал».
- **Подключение**: `ssh username@IP address`

### Стратегия резервного копирования с самого начала
```bash
# Manuelles Backup:
iob backup

# Automatisches Backup mit backitup-Adapter:
# Über Admin-Interface installieren und konfigurieren
```

### Учимся понимать файлы журналов
```bash
# Live-Logs anzeigen:
iob logs --watch

# Spezifische Adapter-Logs:
iob logs adaptername

# System-Logs:
sudo journalctl -u iobroker -f
```

### Базовый подход к поиску и устранению неисправностей
1. **Всегда делайте резервную копию перед внесением изменений.**
2. **Вносите изменения по одному**
3. **Перед тем как экспериментировать бездумно, ознакомьтесь с журналами.**
4. **Для решения сложных проблем используйте форум сообщества.**
5. **Наберитесь терпения — системе нужно время для запуска.**

## Типичные ошибки новичков
### 1. Слишком много изменений, слишком быстро
- **Проблема**: При параллельной установке нескольких адаптеров неясно, что именно вызывает проблемы.
- **Решение**: Проверяйте по одному адаптеру за раз.

### 2. Обновления без резервной копии
- **Проблема**: Система вышла из строя после обновления, восстановление невозможно.
- **Решение**: Выполняйте автоматическое резервное копирование перед каждым крупным обновлением.

### 3. Права root как «решение»
- **Проблема**: Добавление `sudo` перед командами ioBroker не работает.
- **Решение**: Найдите причину, а не пытайтесь силой «исправить» симптом.

### 4. Следуйте устаревшим инструкциям
- **Проблема**: Следование инструкциям из интернета 2018 года.
- **Решение**: Предпочтительнее использовать официальную документацию и актуальные темы на форуме.

### 5. Docker без базовых знаний Linux
- **Проблема**: Использование контейнеров Docker без понимания принципов работы томов, сетей и т. д.
- **Решение**: Сначала основы Linux, затем специализации Docker.

## Как эти основные принципы приводят к устранению неполадок
Понимание этих пяти основных принципов поможет вам гораздо лучше понять следующие главы:

- **"ioBroker больше не работает"**: Устранены проблемы с блокировками базы данных, сбоями в работе административной панели и конфликтами Node.js.
- **«Ошибки адаптеров»**: Рассматриваются проблемы установки, запуска и производительности отдельных адаптеров.

**Важное примечание**: Большинство проблем с ioBroker возникают из-за одного или нескольких упущений в фундаментальных аспектах. Прежде чем приступать к сложным исправлениям, систематически проверяйте эти пять областей.
