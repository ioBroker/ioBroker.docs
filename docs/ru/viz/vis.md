# Визуализации с VIS
## Вообще
@@@ VIS - это мощное приложение, которое заслуживает подробных инструкций и руководств. Форма и местонахождение этого руководства еще не определены.
@@@

## [](https://github.com/ioBroker/ioBroker/wiki/ioBroker-Adapter-vis#konfiguration)
## Конфигурация
Конфигурация адаптера не требуется. Может быть только один экземпляр VIS.

* * *

## [](https://github.com/ioBroker/ioBroker/wiki/ioBroker-Adapter-vis#bedienung)Operation
Адаптер вызывается через `<IPdesServers>:8082/vis/index.html` и отображается визуализация.
Есть несколько просмотров в проекте.
В каждом представлении несколько свободно размещаемых виджетов, которые служат для отображения или работы.
Для создания и настройки визуализации вы должны загрузить **редактор** Редактор вызывается через `<IPdesServers>:8082/vis/edit.html` или по гиперссылке на вкладке «Экземпляры» адаптера администратора.
После закрытия редактора с помощью значка «Закрыть окно» ((x) в правом верхнем углу) последнее отредактированное представление отображается в представлении функций.
Это можно вызвать в любое время с помощью `<IPdesServers>:8082/vis/index.html#ViewName` Редактор разделен на несколько областей.
![обзор](../../de/viz/media/vis_ioBroker_vis_Editor_002-300x165.jpg)

## Гонщики в области головы (1)
### Просмотров
Если выбрана эта вкладка, вы можете выбрать вид для редактирования из существующего выпадающего меню в выпадающем меню ниже.
![iobroker_vis_Editor_Views_Header](../../de/viz/media/iobroker_vis_Editor_Views_Header.jpg) Четыре значка справа говорят сами за себя и относятся к видам.
После нажатия на рабочую область вид на боковой панели «Свойства» справа переходит на вкладку «Виды» и отображает настройки для вида.

### Виджеты
Если активирована вкладка Виджеты, изменяется основная панель инструментов. ![iobroker_vis_Editor_Widgets_Header](../../de/viz/media/iobroker_vis_Editor_Widgets_Header.jpg) Отображаются инструменты для редактирования виджетов.

#### Выпадающее меню
Здесь вы можете выбрать виджет для редактирования, или здесь будет отображаться выбранный виджет.

#### Значок панели
Три значка для удаления, копирования и информации становятся активными, когда виджет выбран.

#### Выровнять виджеты
![](../../de/viz/media/iobroker_vis_Editor_Widgets_Ausrichten_Header.JPG) Если вы выбираете несколько виджетов с помощью нажатой кнопки мыши (или с нажатой клавишей Ctrl), то вы можете выровнять их с помощью этих групп значков (выровненных по левому краю, выровненных по правому краю, по вертикали, по центру, по центру и по вертикали).
Кроме того, вы можете расположить несколько виджетов с одинаковым расстоянием (горизонтальное и вертикальное), а также настроить размер нескольких виджетов с помощью двух последних значков.
При выборе виджета вид на боковой панели «Свойства» справа изменится на вкладку «Виджет» и отобразит настройки для этого виджета.
С возможностью множественного выбора вы можете изменить свойства нескольких виджетов на правой боковой панели одновременно.

#### Все виджеты
Два виджета используются для облегчения редактирования видов.
Первый гарантирует, что значения виджетов не обновляются, второй предотвращает перемещение виджетов.

#### Экспорт виджета
Если виджет активирован, вы можете использовать эту кнопку, чтобы экспортировать код CSS виджета.

#### Виджет импорта
И наоборот, здесь можно, например, Импортируйте виджеты, которые не входят в область видимости.
После нажатия на виджет вид на боковой панели «Свойства» с правой стороны изменится на вкладку «Виджет» и отобразит настройки для выбранного виджета.

### Инструменты
Если активирована вкладка «Инструменты», меняется основная панель инструментов.

#### Разрешение
Здесь много предустановленных разрешений экрана, отображаемых на мобильных устройствах, но также пункт «пользовательский» предлагает возможность сделать свои собственные настройки.
Если вы выберете что-то здесь, на рабочем столе будет нарисована рамка, соответствующая выбранному разрешению экрана.

#### По умолчанию
Если установлен флажок _default_, активный [вид](http://www.iobroker.net/?page_id=1193&lang=de) становится видом по умолчанию и загружается, если вид VIS (см. [проект](http://www.iobroker.net/?page_id=188&lang=de&preview_id=188&preview_nonce=d845a20ee2&preview=true#Projekt)) загружается с разрешением, установленным выше.
** Пример: ** Дисплей _iPad Portrait_ выбран, и поле _Default_ отмечено в представлении _Start_.
Предполагая, что проект теперь загружает представление с именем _Weather_ в альбомном формате на iPad.
Если iPad теперь повернут в портретную ориентацию, автоматически отобразится представление _Start_, ранее определенное для этого разрешения (портрет) по умолчанию (_default_).
Например, «возврат к главному меню» или _hoch_, показывающий вид, отличный от _quer_, возможен с соответственно перемещаемыми устройствами.

#### Сетка
Вот возможности _inaktiv_, _Elemente_ и _Raster_. Если вы выбираете элементы, виджеты привязываются при перемещении мыши к соседнему элементу. С помощью Raster вы можете указать размер сетки (в пикселях), в которой виджеты будут вставлены на место.

#### ID экземпляра
Здесь отображается уникальный идентификатор, например, через VIS [Команда управления](https://github.com/iobroker/ioBroker.vis/blob/master/README.md#control-interface) можно адресовать с помощью скрипта.

#### Идентификатор браузера
Если вы нажмете эту кнопку, новый идентификатор будет создан в предыдущем поле. Это делает каждый браузер индивидуально идентифицируемым на каждом устройстве.

Экспорт ####
Здесь вы можете экспортировать свой вид, например использовать в другой установке. При нажатии на кнопку открывается окно, в котором вид предлагается в текстовом виде. Эти данные могут быть помещены в буфер обмена с помощью Ctrl-C и загружены в редактор, а затем сохранены

Импорт ####
После нажатия открывается пустое окно. Здесь вы можете скопировать текстовый файл, полученный с помощью функции _export_. Затем введите имя для представления в левом нижнем углу и нажмите кнопку _importieren_. Оттуда, новый вид доступен. Эта функция полезна для импорта примеров с форума.

#### Проекты
Проекты являются коллекциями [мнения](http://www.iobroker.net/?page_id=1193&lang=de).
По умолчанию проект _main_ создается и вызывается через `<IPdesServers>:8082/vis/index.html#ViewName`.
Файлы проекта находятся в установке ioBroker в папке `_ioBroker-Ordner_/iobroker-data/files/vis.0/main`.

#### Проект экспорт / импорт
Проекты можно экспортировать целиком (например, для совместного использования с другими пользователями): создается zip-файл, содержащий используемые изображения, таблицу стилей vis-user.css и фактические определения vis-views.json.
Кроме того, есть возможность экспортировать проект анонимно. : construction: Чтобы импортировать другие проекты, просто перетащите zip-файл экспорта, описанного выше, в окно и введите имя для нового проекта: ![](../../de/viz/media/iobroker_vis_Editor_Setup_Projekt_ImportHeader.gif)

#### Новый проект ...
Здесь вы можете создать новый проект.
Откроется окно с просьбой ввести имя проекта.
После подтверждения редактор загружает новый проект по этому адресу: `<IPdesServers>:8082/vis/_projektname_/index.html#ViewName.` Файлы нового проекта можно найти в установке ioBroker в `_ioBroker-Ordner_/iobroker-data/files/vis.0/projektname`.
** Совет **: Вы можете дублировать папку проекта и редактировать копию в VIS Editor `<IPdesServers>:8082/vis/_projektname_duplikat_/index.html`.

#### Приложение
При загрузке проекта все [Views]: construction: (http://www.iobroker.net/?page_id=1193&lang=en) этого [project]: construction: (http://www.iobroker.net/?page_id = 188 & lang = DE & preview_id = 188 & preview_nonce = d845a20ee2 & preview = true # project).
Иногда это не имеет смысла и замедляет работу системы.
Поэтому можно суммировать взгляды, которые принадлежат друг другу в разных проектах.
Имеет смысл, например, разделение устройств с разными разрешениями экрана.
Таким образом, вы можете создать проект _Smartphone_ и создавать виды, оптимизированные для сенсорного управления и вертикального выравнивания.
Из-за преднамеренного сокращения виджетов проект остается небольшим по объему данных и, следовательно, быстро загружается в пути (например, через VPN и портативное радио).
Другой проект _Wandtablet_, который отображается только на планшете на стене, не требует загрузки представлений из проекта _Smartphone _.
Основной проект _main_ для отображения всех данных обычно отображается и редактируется только с настольного ПК.
Здесь объем данных и скорость вычислений обычно не имеют значения.
Если представления должны использоваться в нескольких проектах, их можно скопировать с помощью команды [экспорт](http://www.iobroker.net/?page_id=188&lang=de&preview_id=188&preview_nonce=d845a20ee2&preview=true#exportieren): construction: / [импорт](http://www.iobroker.net/?page_id=188&lang=de&preview_id=188&preview_nonce=d845a20ee2&preview=true#importieren): construction :.
Связывание через навигационные виджеты из одного проекта в другой невозможно.
** Совет **: Также может быть полезно создать тестовый проект при экспериментировании с командами CSS.

#### Настройки ... (7)
![](../../de/viz/media/iobroker_vis_Editor_Setup_Projekteinstellungen.JPG)

* _ **Перезагрузите, если нет более длинного соединения, чем:** _ Активное представление полностью перезагружается,

если соединение между интерфейсом (планшетом) и сервером было прервано на более длительное время, чем заданное время.
Чтобы предотвратить это, это время можно также установить на _ **never** _.

* _ **интервал переподключения:** _ Время пробовать с внешнего интерфейса

добраться до сервера.

* _ **Экран повторного подключения в темноте:** _ Обычно при попытке перезагрузить страницу отображается белый цвет.

Чтобы не беспокоить это в темных помещениях, этот флажок можно переключить на темный экран.

* _ **Удалить из ОЗУ неактивные представления:** _ Чтобы сохранить ценную память на внешнем интерфейсе,

который обычно составляет всего 1 ГБ для недорогих планшетов, представления, которые больше не нужны, могут быть удалены из ОЗУ.
Однако при перезагрузке соответствующего представления это занимает больше времени.
Эта опция устанавливает время, в течение которого неиспользуемые представления должны храниться в оперативной памяти.

## Боковая панель виджета (2)
Используется для выбора виджетов.
Виджеты отображаются в виде значков и могут быть перетащены на рабочий стол или расположены с помощью кнопки _Insert_ в позиции 0.0 рабочего стола.
![iobroker_vis_Editor_Widgets_sidebar](../../de/viz/media/vis_iobroker_vis_Editor_Widgets_sidebar.jpg) Поле под кнопкой _Insert_ является полем фильтра.
Здесь вы можете ввести термин для поиска значков впоследствии.
Все значки, которые содержат этот термин отображаются.
Если вы удалите последний фильтр (или \ *), вы получите выпадающий список с возможными условиями поиска.
Ниже поле выбора для наборов виджетов.
Звездочка (*) обозначает все наборы виджетов.
При фильтрации по терминам все наборы виджетов автоматически ищутся.
В противном случае, однако, выпадающее меню также предлагает различные наборы виджетов в качестве фильтров.

## Боковая панель настроек (4)
Здесь все настройки для видов и виджетов вводятся через соответствующие вкладки.
Кроме того, вкладка CSS предлагает опытным пользователям возможность интегрировать свои собственные разработки.

* Назначение точки данных
* Размер виджета
* Размер шрифта и цвет
* Фон
* Рамка, цвет линии, стиль, толщина
* параметр css

Для отображения точек данных или выполнения действий точка данных должна быть назначена виджету.
Эту запись можно найти в разделе _General_.
