**Dieser Adapter verwendet Sentry-Bibliotheken, um Ausnahmen und Codefehler automatisch an die Entwickler zu melden.** Weitere Details und Informationen zum Deaktivieren der Fehlerberichterstattung finden Sie in Abschnitt [Sentry-Plugin-Dokumentation](https://github.com/ioBroker/plugin-sentry#plugin-sentry)! Die Sentry-Berichterstattung wird ab js-controller 3.0 verwendet.

Adapter für den Zugriff auf die Spotify-Wiedergabesteuerung. Aufgrund der Spotify-API ist ein Premium-Abo erforderlich.

Verbindung zu [Spotify Premium API](https://www.spotify.com/).

## Dokumentation
Siehe auch [Spotify Developer API-Dokumentation](https://developer.spotify.com/).

### Einrichtung / Autorisierung
1. Melden Sie sich auf https://developer.spotify.com/dashboard/ an.
2. Erstellen Sie eine Anwendung. Sie erhalten eine Client-ID und ein Client-Geheimnis (siehe [Anleitung](docs/create_app.png)).
3. Legen Sie in den App-Einstellungen Ihrer erstellten Spotify-Anwendung die Umleitungs-URIs auf `https://oauth2.iobroker.in/spotify` fest.
4. Geben Sie die Client-ID und das Client-Geheimnis in die unten stehenden Felder ein.
5. Starten Sie die Instanz.
6. Wechseln Sie zur Registerkarte „Objekt“ und klicken Sie auf die Schaltfläche „getAuthorization“ unter „spotify-premium.0.authorization“.
7. Kopieren Sie die angezeigte URL von `spotify-premium.0.authorization.authorizationUrl` in Ihren Webbrowser und rufen Sie sie auf.
8. Möglicherweise müssen Sie sich bei Spotify anmelden und den Zugriff gewähren.
9. Der Browser wird auf eine ungültige URL umgeleitet. Falls der Fehler „ungültige Umleitungs-URI“ auftritt, überprüfen Sie bitte Schritt 3.
10. Kopieren Sie diese URL und fügen Sie sie in `spotify-premium.0.authorization.authorizationReturnUri` ein.
11. Der Wert in `spotify-premium.0.authorization.authorized` wird auf „true“ gesetzt, wenn alles erfolgreich war.

Wir müssen eine HTTPS-Weiterleitungs-URI angeben, da Spotify ab Dezember 2025 keine unsicheren URIs mehr zulässt.
