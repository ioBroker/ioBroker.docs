### IoBroker.teltonika
<img width="100" height="100" src="https://raw.githubusercontent.com/ioBroker/ioBroker.teltonika/refs/heads/main/admin/teltonika.svg" />

**类别：** 网络 **开发者：** bluefox

此适配器支持通过高效的 MQTT 通信将 Teltonika 路由器连接到 ioBroker。

Teltonika 路由器常用于专业和半专业环境，例如 LTE/5G 备份、站点联网、远程维护或边缘物联网应用。

此适配器可将路由器提供的状态和遥测数据无缝集成到 ioBroker 控制面板、监控设置或自动化流程中。

该适配器支持多种路由器型号，包括 **RUT**、**TRB**、**OTD**、**RUTM** 和 **RUTC** 系列，并提供丰富的设备数据作为状态。

#### 采集的数据点（取决于模型）：
- 温度（`RUT2`、`RUT9`、`RUTX`、`RUT3`、`RUT1`、`TRB2`、`TRB5`、`OTD`、`RUTM`、`RUTC`）
- 信号强度和移动网络提供商
网络状态和连接类型（2G/3G/4G）
- WAN IP 地址
- 正常运行时间
- 设备名称
- 数字输入（`RUT9`：输入 1/2）
- 模拟输入（`RUT9`、`TRB2`、`TRB141`）
- 引脚状态（引脚 2、3、4 – 因型号而异）

#### 特色功能：
- 基于 MQTT 的快速、资源高效的通信
- 使用专用端口 **1885** 以避免与其他 MQTT 实例冲突
- **轻松上手：**只需在路由器中激活 MQTT 并将其指向 ioBroker 代理即可。
- **已使用 RUTC 系列设备进行测试**，兼容所有支持 MQTT 的设备
- **Sentry 集成**，用于自动错误检测（从 js-controller 3.0 开始）
- 非常适合网络监控、日志记录、警报和自动化场景

🔗[适配器的 GitHub 页面](https://github.com/ioBroker/ioBroker.teltonika)

### IoBroker.serial-gps
<img width="100" height="100" src="https://raw.githubusercontent.com/ioBroker/ioBroker.serial-gps/refs/heads/main/admin/serial-gps.svg" />

**开发者：** `bluefox` **类别：** 实用工具

此适配器可读取通过**USB**或**串口**连接的GPS鼠标的GPS数据。这使得位置数据（例如，车辆、户外或移动智能家居设备的位置数据）能够直接集成到ioBroker中。该适配器尤其适用于地理围栏自动化、位置跟踪、运动分析或基于时间和位置的操作等场景。

支持提供标准化**NMEA数据集**的典型GPS接收器，例如许多GPS模块、USB天线或便携式接收器的输出。

#### 已记录数据：
该适配器会处理以下 GPS 信息：

- 纬度和经度
- 速度
- 高度
卫星信息
航向/方向
- UTC时间
- 准确度值（取决于模型）

数据会不断被读取，并以结构清晰的 GPS 状态形式在对象树中提供。

#### 特色功能：
- 支持提供 NMEA 语句的 **USB 和串行 GPS 接收器**
- 可在多种平台上运行，包括 Raspberry Pi 和基于 Linux 的系统
- 非常适合具有位置分析功能的移动项目或设置
- 无需网络连接——纯本地GPS定位
- 根据设备时钟速度提供持续的位置更新
- 非常容易上手：选择 COM 端口/USB 端口即可立即开始使用。
- 包含用于错误分析的 Sentry 集成（从 js-controller 3.0 开始）

🔗 [适配器的 GitHub 页面](https://github.com/ioBroker/ioBroker.serial-gps)

该适配器能够将车辆信息可视化、评估并集成到自动化系统中（例如，当汽车到达时打开车库门）。

该适配器提供多种数据点，包括：

车辆位置
- 里程
- 燃油量/电池状态
- 充电状态和充电过程（适用于电动/混合动力车型）
- 车门、车窗、后挡板的状态
- 锁定状态
- 维护和服务信息
室外温度
车辆类型、车辆识别码和型号
- 通过 MQTT 进行实时事件（取决于型号和 API）

#### 特色功能：
- 使用官方的**BMW CarData API**
- 现代 OAuth2 身份验证
- **MQTT实时流**，用于实时数据传输
- 支持同一账户下拥有多辆车
- 非常适合用于位置自动化（“地理围栏”）、智能充电或家庭集成
所有数据点在 ioBroker 对象树中都结构清晰。

#### 根据 README 文件中的说明：
- 需要一个已关联车辆的宝马账户。
- API 的行为可能因车辆型号和在线套餐而异。
- 该适配器定期检索数据，并在必要时补充实时流。

#### 支持的数据和函数：
- OMV服务器的系统信息
- 驱动器和文件系统状态
- SMART 信息（取决于模型）
- CPU、RAM 和一般系统数据
- 服务状态（取决于 OMV 配置）
- 对象树中数据点的自动结构映射

#### 特色功能：
- 通过 OpenMediaVault 的原生 RPC API 进行连接
- OMV 不需要任何额外的软件或插件
- 非常适合集中监控 NAS 或家庭实验室服务器
- 支持 OMV 5/6（取决于 API 兼容性）
- 清晰的数据结构，便于仪表盘和分析

SRM 是 Synology 路由器（例如 RT6600ax、RT2600ac 和 MR2200ac）的操作系统。它提供专业的网络功能，例如 VLAN、家长控制、流量分析器、VPN 服务器、Wi-Fi Mesh 等等——所有这些都可以通过类似于 Synology DSM NAS 操作系统的现代化 Web 界面进行管理。

该适配器能够从群晖路由器检索关键系统数据，并以结构化的方式将其呈现在 ioBroker 对象树中。这使其成为监控家庭网络或根据网络状态或客户端活动做出响应的自动化操作的理想选择。

#### 支持的数据和函数：
根据README文件，其中包含以下信息：

- 路由器型号和固件版本
- 系统信息（CPU 负载、内存使用情况）
网络状态
- 广域网/局域网信息

## 适配器更新和功能
### IoBroker.ping – 新功能：TCP 端口检查
<img width="100" height="100" src="https://raw.githubusercontent.com/ioBroker/ioBroker.ping/master/admin/ping.png?raw=true" />

广受欢迎的 **ioBroker.ping** 适配器已得到增强，现在除了传统的 ICMP ping 之外，还可以检查 **TCP 端口**。

这使您不仅可以确定设备是否可达，还可以确定目标系统上是否正在运行特定的 **服务**，例如 Web 服务器、MQTT 代理或 SSH 服务。

TCP端口检查有什么好处？
借助这项新功能，适配器现在可以：

- 检查特定端口是**打开**还是**关闭**
- 确定某项服务是否**可用**或**已停止服务**
- 提高依赖网络的自动化流程的可靠性
- 提供比传统 ping 更精确的状态更新

例如：主机响应 ping 请求，但端口 1883 (MQTT) 关闭 → 表明服务出现问题。

#### 典型应用场景：
- 检查远程系统的 **ioBroker** 端口是否可达
- 监控网络服务器（端口 80/443）
- 监控 SSH（端口 22）
- 对数据库、网关、智能家居设备进行可用性检查
- 服务失败时触发操作

#### 特色功能：
- 除了普通的 ping 功能外，还可以同时使用两者。
系统负载极低
- 配置简单：输入 IP/主机名 + 端口
- 非常适合家庭网络、服务器、NAS 和智能家居基础设施

🔗 [GitHub](https://github.com/ioBroker/ioBroker.ping)
