# IoBroker 月度回顾 – 2026 年 1 月/2 月
ioBroker 的新年伊始就迎来了重大更新！在 2026 年的前两个月里，我们向 `latest` 仓库添加了众多新的适配器，其中一些已经升级到 `stable` 版本。

此外，Z-Wave 用户也有重要消息：之前的 `zwave2` 适配器已被更现代化的解决方案所取代。

## `latest`中的新适配器
这些适配器在发布稳定版本之前，欢迎测试人员、错误报告和改进建议。

### IoBroker.hagelschutz-vkf
<img width="100" height="100" src="https://raw.githubusercontent.com/UncleSamSwiss/ioBroker.hagelschutz-vkf/master/admin/hagelschutz-vkf.jpg" />

**类别：** 天气/建筑防护 **开发商：** UncleSamSwiss

此适配器将**州建筑保险协会 (VKF)**提供的**hagelschutz-einfach-automatisch.ch**服务集成到 ioBroker 中。

该服务可自动发出冰雹预警，使遮阳篷、卷帘门和其他敏感的户外设施能够及时收起。

#### 特色功能：
- 数据来自瑞士官方VKF冰雹预警服务
- 非常适合在冰雹危险情况下自动收回遮阳篷和遮阳伞。
- **仅限瑞士地区使用**（使用VKF服务）
配置非常简单

**注意：**该适配器名为`hagelschutz-vkf`，因为原名称`hagelschutz`已被占用。

### IoBroker.foxesscloud
<img width="100" height="100" src="https://raw.githubusercontent.com/inventwo/ioBroker.foxesscloud/master/admin/foxesscloud.png" />

**类别：**能源/太阳能 **开发商：**inventwo

该适配器通过 **FoxESS 云 API** 从 **FoxESS 太阳能逆变器** 获取实时数据。这使得太阳能系统、电池储能和并网发电能够方便地集成到 ioBroker 中。

FoxESS 是一家广泛使用的太阳能逆变器制造商，其产品也应用于 **Enpal** 太阳能发电厂。

#### 记录的数据点：
- `pvPower` – 当前光伏发电量（千瓦）
- `generationPower` – 总输出功率 (kW)
- `load` – 当前功耗（千瓦）
- `gridConsumption` – 电网消耗量（千瓦）
- `feedinPower` – 并网功率 (kW)
- `soc` – 电池电量 (%)
- `batCharge` / `batDischarge` – 电池充电/放电功率 (kW)
- 连接状态

#### 特色功能：
- 每天最多 **1440 次 API 调用**（每分钟 1 次调用）
- 支持 FoxESS 和 Enpal 系统
- 非常适合用于能源管理、能耗分析和自动化
清晰的数据点结构

🔗 [适配器的 GitHub 页面](https://github.com/inventwo/ioBroker.foxesscloud)

### IoBroker.apsystems-ez1
<img width="100" height="100" src="https://raw.githubusercontent.com/zhihaining/ioBroker.apsystems-ez1/master/admin/apsystems-ez1.png" />

**类别：**能源/太阳能 **开发商：**智海宁

该适配器通过设备的本地 HTTP API（端口 8050）集成 **APsystems EZ1 微型逆变器**，无需云，无需外部依赖。

APsystems EZ1 是紧凑型微型逆变器，可直接连接到单个太阳能组件，特别适用于**阳台发电厂**和小型光伏系统。

支持的函数：
- 实时性能和能耗值（每个通道和总计）
- 设备信息（固件、SSID、IP 地址）
- 查询报警状态
- 设置**最大功率**（限制输出功率）
- 开关设备**开/关**
- 一个实例中的多个设备

#### 特色功能：
完全本地化，无需依赖云端
- HTTP 超时和自动重试逻辑
- 可选的**重复错误电子邮件通知**
- 包含一个用于概览和控制的 **VIS2 小部件**

🔗 [适配器的 GitHub 页面](https://github.com/zhihaining/ioBroker.apsystems-ez1)

### IoBroker.solectrus-influxdb
<img width="100" height="100" src="https://raw.githubusercontent.com/patricknitsch/ioBroker.solectrus-influxdb/master/admin/solectrus-influxdb.png" />

**类别：**能源/数据库 **开发者：**patricknitsch

此适配器将选定的 ioBroker 状态写入 **InfluxDB 2.x** 数据库，并辅以强大的公式引擎生成衍生值。

它专为能源监控系统（光伏系统、电池储能、热泵、壁挂式储能箱、电网计量）而开发。

主要功能：
- **传感器映射：** 将任何 ioBroker 状态配置为 InfluxDB 测量值和字段（int、float、bool、string）
- **可靠缓冲区：**持久写入缓冲区（最多 100,000 个数据点），即使 InfluxDB 服务中断也能正常运行
- **公式引擎：**根据多个来源计算值（例如，自用率、自覆盖率）
- **状态机模式：** 来自基于规则的条件的字符串或布尔状态
- **可视化公式生成器：** 拖放式、实时预览、运算符提示
- 计算值的文件夹分组

#### 特色功能：
- 专为与 **SOLECTRUS** 配合使用进行能源监测而设计
- 通过持久缓冲来应对数据库故障
- 公式引擎无需外部脚本适配器即可进行复杂的计算。

🔗 [适配器的 GitHub 页面](https://github.com/patricknitsch/ioBroker.solectrus-influxdb)

- 温度和湿度读数（Sense）
- 水流量、压力值和泄漏检测（Sense Guard）
- 阀门控制（开/关）
- 消费统计数据
- 错误和警报消息

#### 特色功能：
- **OIDC/Keycloak 认证** – 刷新令牌以加密形式存储。
- 可配置的查询间隔
- 完整的德语和英语文档
- 非常适合智能家居中的漏水防护自动化

🔗 [适配器的 GitHub 页面](https://github.com/patricknitsch/ioBroker.grohe-smarthome)

### IoBroker.meteoswiss
<img width="100" height="100" src="https://raw.githubusercontent.com/deMynchi/ioBroker.meteoswiss/master/admin/meteoswiss.png" />

**类别：**天气 **开发者：**deMynchi

该适配器为 ioBroker 免费提供来自瑞士联邦气象和气候局 MeteoSwiss 的天气数据，无需 API 密钥。

MeteoSwiss 是瑞士的官方气象服务机构，提供高度精确的天气数据和针对瑞士地形量身定制的预警信息。

#### 支持的数据：
- 当前天气及短期天气预报
- **每3小时一次的天气预报**（包括汇总的最低/最高/平均值）
- **恶劣天气预警**，包含严重程度信息：冰雹、暴风雨、降雨、降雪等。
- 每 10 分钟自动更新（间隔根据数据源调整）
- 明确区分：对于未知数据，使用 `null` 值而不是 null。

#### 特色功能：
- **无需 API 密钥**，可直接使用
- 支持同时显示多个警告类别（显示最重要的警告类别）
- 非常适合需要特定地点天气信息的瑞士用户
- 可以根据恶劣天气预警实现自动化（例如，卷帘门自动收回）

### IoBroker.autodarts
<img width="100" height="100" src="https://raw.githubusercontent.com/inventwo/ioBroker.autodarts/master/admin/autodarts.svg" />

**类别：**娱乐/智能家居 **开发商：**inventwo

此适配器将 ioBroker 连接到 **Autodarts Board Manager**——这款广受欢迎的电子飞镖靶系统的本地运行服务器。

Autodarts 允许您使用摄像头和软件将传统的飞镖靶改造成智能的自动识别系统。

此适配器允许将游戏事件直接集成到 ioBroker 自动化中：例如，击中靶心时点亮指示灯、通过 TTS 宣布下一枪、或记录统计数据。

#### 记录的数据点：
- 得分和当前记录（`visit.score`，`throw.current`）
- 特殊命中触发：正中、三连击、双连击、未命中
- 棋盘状态（交通灯颜色：绿色 = 投掷飞镖，黄色 = 取下飞镖，红色 = 离线）
- 硬件：电路板照明和电源可控
系统信息：软件版本、CPU、摄像头配置
- 支持游戏公告的文本转语音功能

#### 特色功能：
- 连接到本地 Autodarts 棋盘管理器（标准端口 3180）
- 无需依赖云端 – 完全本地化
- 非常适合用于自动化游戏事件（灯光、声音、场景）
- 针对靶心、三连区、双连区或任何飞镖投掷的触发式动作

🔗 [适配器的 GitHub 页面](https://github.com/inventwo/ioBroker.autodarts)

### IoBroker.zwavews
<img width="100" height="100" src="https://raw.githubusercontent.com/arteck/ioBroker.zwavews/master/admin/zwavews.png" />

**类别：**协议/智能家居 **开发商：**arteck

**ioBroker.zwavews** 适配器是之前的 `zwave2` 适配器的现代继任者，它通过 WebSocket 经由 **zwave-js-ui** 实例与 Z-Wave 设备建立连接。

**重要信息：** 之前的 `zwave2` 适配器已于 2026 年 2 月从 `stable` 存储库中移除。请用户迁移到 `zwavews`。

Z-Wave 是一种成熟的无线协议，专为智能家居应用而设计。

它工作在 GHz 以下频段（欧洲为 868 MHz），为 Wi-Fi 和 Zigbee 提供了一种可靠、无干扰的替代方案，支持网状网络，且能耗低。

支持的函数：
- 通过 Home Assistant WebSocket API 连接到 **zwave-js-ui**
- 根据 ioBroker 的状态自动创建所有 Z-Wave 设备对象
- 启动后，启用 `info.sendMessageAllowed` 状态以进行受控消息管理
- 轻松迁移：从 `zwave2` 复制缓存文件，然后唤醒一次电池供电设备。

#### `zwave2` 的迁移：
1. 将旧适配器中的缓存文件复制到新的 `zwave-js-ui`。
2. 唤醒电池供电设备一次（按一下按钮）
3. 配置并启动 `zwavews` 适配器

🔗 [适配器的 GitHub 页面](https://github.com/arteck/ioBroker.zwavews)

### IoBroker.mitsubishi-local-control
<img width="100" height="100" src="https://raw.githubusercontent.com/Black-Thunder/ioBroker.mitsubishi-local-control/master/admin/mitsubishi-local-control.png" />

**类别：** 气候控制 **开发商：** Black-Thunder

该适配器通过直接本地连接集成**三菱电机空调**——无需云端，无需外部服务。

三菱电机是家用和商用分体式空调和热泵的领先制造商之一。**WLAN接口**允许通过家庭网络对系统进行本地控制。

支持的函数：
- 设置**运行模式**（制冷、制热、烘干、通风、自动）
- 读取并写入**目标温度**
- **调节风扇转速**
- **控制百叶窗/气流方向**
- 自动同步设备状态
- 一个实例中的多个设备

#### 特色功能：
完全本地化——无需依赖云端
- 通过三菱WLAN接口（MELCloud模块或兼容模块）直接访问
- 按可配置的时间间隔自动查询状态
- 完整的德语和英语文档

🔗 [适配器的 GitHub 页面](https://github.com/Black-Thunder/ioBroker.mitsubishi-local-control)
