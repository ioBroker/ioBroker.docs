夏季即将结束，过去几个月 ioBroker 社区可谓风起云涌。

众多全新适配器已发布；部分适配器已加入稳定版软件仓库，其余适配器则已上线最新版软件仓库，等待您进行测试。

让我们详细了解一下 2025 年 9 月的亮点。

## 稳定版新增适配器
### **ioBroker.vis-2-widgets-icontwo**
<img src="https://raw.githubusercontent.com/inventwo/ioBroker.vis-2-widgets-icontwo/main/admin/vis-2-widgets-icontwo.png" width="100" height="100" />

**类别：**可视化/图标

**开发者：** skvarel

此软件包为 VIS 2 添加来自 inventwo 的图标集。

这些图标专为智能家居场景优化，可快速提升可视化效果。

新增功能是这些图标现在可以直接在 VIS-2 图标选择器中使用，从而可以非常轻松地将其插入到仪表板中。仅支持 VIS-2 2.13.x 及更高版本。

最近的版本添加了许多新符号，包括水（洒水器、瀑布）、门窗、电池存储、天气方向、地图和人物等符号。

**提示：**安装后，您应该重启电脑并清除浏览器缓存，以便所有图标都能正确显示。

### **ioBroker.brightsky**
<img src="https://raw.githubusercontent.com/ticaki/ioBroker.brightsky/main/admin/brightsky.png" width="100" height="100" />

**类别：**天气

**开发者：** ticaki

Brightsky 适配器将 Bright Sky 天气服务集成到 ioBroker 中。

该 API 提供来自德国气象局 (DWD) 的天气数据，并以易于使用的 JSON 格式呈现。

**特色功能：**

- 数据每 30 分钟更新一次，并且始终来自最近的 DWD 气象站。
- 如果一个站点没有提供任何值，适配器会自动回退到下一个最近的站点或更远的站点（回退系统）。
除了温度、降水、湿度和风等经典天气数据外，API 还提供辐射值（太阳能），这对光伏运营商来说尤其有趣。
- 该 API 可免费访问，无需 API 密钥，并且完全基于开源。

链接到 [适配器的 GitHub 页面](https://github.com/ticaki/ioBroker.brightsky)

### **ioBroker.pirate-weather**
<img src="https://raw.githubusercontent.com/ticaki/ioBroker.pirate-weather/main/admin/pirate-weather.png" width="100" height="100" />

**类别：**天气

**开发者：** ticaki

Pirate Weather 是一个开源天气 API，是为替代已停用的 Dark Sky 而开发的，它基于 NOAA 数据。

该适配器可将数据直接导入ioBroker。对于希望使用可靠天气信息来驱动自动化流程的用户来说，这是理想之选。

**GitHub 上的详细信息：**

- 使用需要 API 令牌，可通过 `docs.pirateweather.net` 获取。
- 该适配器支持 Pirate-Weather API v2，并提供温度、湿度、降水、风和月相数据。
湿度、云量、降水概率和月相等值直接以百分比值输出。
- 风向或摘要等文本会自动适应 ioBroker 的系统语言。
- 轮询间隔可以以分钟为单位进行配置。错误会被捕获，然后适配器会按照固定的时间表重启，而不会在日志中造成无限循环。

链接到 [适配器的 GitHub 页面](https://github.com/ticaki/ioBroker.pirate-weather)

### **ioBroker.pi-hole2**
<img src="https://raw.githubusercontent.com/oweitman/ioBroker.pi-hole2/main/admin/pi-hole2.png" width="100" height="100" />

**类别：**网络

**开发者：** weweitman

该适配器可将 Pi-hole（版本 6 及更高版本）集成到 ioBroker 中。

这样，您可以监控广告拦截器的统计数据、请求和拦截率，并将其用于自动化操作。

**特色功能：**

- 专门支持 Pi-hole v6（对于旧版本，请继续使用 ioBroker.pi-hole 适配器）。
- 可以通过 ioBroker 直接屏蔽或解除屏蔽域名，包括在指定时间后自动重新激活。
- 统计信息和版本信息以数据点的形式存储，可以选择启用/禁用。
- 查询间隔可自由配置——从 1 秒到 24 小时。
- 可以通过 sendTo() 函数直接向 Pi-hole 发送命令。

这意味着该适配器不仅适用于监控，还适用于通过自动化主动控制 Pi-hole。

链接到 [适配器的 GitHub 页面](https://github.com/oweitman/ioBroker.pi-hole2)

### **ioBroker.energy-tracker**
<img src="https://raw.githubusercontent.com/energy-tracker/ioBroker.energy-tracker/main/admin/energy-tracker.png" width="100" height="100" />

**类别：**能源管理

**开发者：** 能源追踪器

此适配器可将电表读数自动传输至能源追踪平台。

这大大简化了能源消耗或并网数据的记录工作。

**GitHub 上的详细信息：**

- 要使用此服务，需要注册 Energy Tracker 帐户并创建个人访问令牌。
- 任何 ioBroker 状态都可以定义为一个源（例如，电表、燃气表或水表）。
- 这些值通过平台的公共 REST API 传输到相应的设备 ID。
发送前可以选择对数值进行四舍五入。
- 该适配器以计划模式运行：必须在 ioBroker 中配置计划，以指定何时传输数据。如果没有计划，则不会发送任何值。
- 令牌以加密形式存储，通信是单向的——只有值被传输到平台，没有数据被读取回来。

链接到 [适配器的 GitHub 页面](https://github.com/energy-tracker/ioBroker.energy-tracker)

<img src="https://raw.githubusercontent.com/quorle/ioBroker.smartloadmanager/main/admin/smartloadmanager.png" width="100" height="100" />

该适配器用于根据光伏发电盈余情况对负载进行动态控制。其目的是在本地消耗多余的能量，避免或完全阻止将其输送到公共电网。

灵活的超时和时间表可确保设备不会频繁地开关机。
- 在当前版本（0.0.7）中，超时函数中的错误已修复，配置也得到了改进。

### **ioBroker.xsense**
<img src="https://raw.githubusercontent.com/arteck/ioBroker.xsense/main/admin/xsense.png" width="100" height="100" />

**类别：**物联网系统/安全

**开发者：** arteck

该适配器可将 X-Sense 设备（包括烟雾、一氧化碳、热量和水浸探测器以及湿度计）集成到 ioBroker 中。通信通过 X-Sense 云平台进行；需要 X-Sense Bridge SBS50 和 X-Sense 帐户。

**值得一提的是：**

- 使用注意事项：频繁轮询（默认 5 分钟）会主动唤醒设备，并可能缩短电池寿命；该适配器并非设计为报警系统，主要用于状态/电池监控。
- 登录限制：应用程序和第三方软件无法同时登录。建议使用第二个帐户，并将主帐户中的设备共享到该帐户；这些登录凭据需输入到适配器中。
- 需要 Python：该适配器使用 Python 环境（在 Linux/Docker 下自动安装，Windows 下需要安装 Python 并根据需要指定路径）。故障排除信息请参阅 README 文件。
- 便利性：手动更新的“强制刷新”按钮；版本中的各种修复/改进（例如角色、时间状态）。

链接到 [适配器的 GitHub 页面](https://github.com/arteck/ioBroker.xsense)

### **ioBroker.steam**
<img width="100" height="100" src="https://raw.githubusercontent.com/bloop16/ioBroker.steam/main/admin/steam.png" />

**类别：** 多媒体/游戏

**开发者：** bloop16 / Martin Rauscher

该适配器将来自 Steam API 的信息集成到 ioBroker 中。

这使您可以监控用户配置文件和游戏，并将其用于自动化操作。

**特征：**

- 个人资料信息：玩家名称、头像、状态（在线、游戏内、离开）、SteamID64。
- 游戏监控：
- 可定义的游戏列表，包括应用 ID。
- 自动检测已播放曲目。
- 一键导入您所有的 Steam 游戏。
- 显示游戏图标、徽标、社区链接和游戏时间统计信息。
- 游戏新闻每6小时更新一次。
- 提示功能：如果出现拼写错误，适配器会从 Steam 数据库中推荐最多 5 款类似游戏。
- API 管理：
- 玩家查询间隔可自由调节（最短 15 秒，标准 60 秒）。
- 系统会自动监控每天最多 10,000 次 API 请求的限制。
查询经过优化，可避免重复请求。

这样一来，玩家就可以直接在智能家居系统中查看他们的Steam活动或游戏统计数据。例如，设置“在线”通知或记录游戏时间。
