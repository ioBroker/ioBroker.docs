# Ежемесячный обзор ioBroker – январь/февраль 2026 г.
Новый год в ioBroker начался с размахом! За первые два месяца 2026 года в репозиторий `latest` было добавлено множество новых адаптеров, а некоторые из них уже обновлены до версии `stable`.

Также есть важная новость для пользователей Z-Wave: предыдущий адаптер `zwave2` был заменен более современным решением.

## Новые адаптеры в `latest`
К разработке этих адаптеров приветствуются отзывы тестировщиков, сообщения об ошибках и предложения по улучшению, прежде чем они будут выпущены в качестве стабильных версий.

### IoBroker.hagelschutz-vkf
<img width="100" height="100" src="https://raw.githubusercontent.com/UncleSamSwiss/ioBroker.hagelschutz-vkf/master/admin/hagelschutz-vkf.jpg" />

**Категория:** Защита от непогоды / Защита зданий **Застройщик:** UncleSamSwiss

Этот адаптер интегрирует сервис **hagelschutz-einfach-automatisch.ch** от **Ассоциации кантональных страховщиков зданий (VKF)** в ioBroker.

Сервис автоматически предупреждает о граде, позволяя своевременно убрать навесы, рольставни и другие чувствительные наружные конструкции.

#### Особые функции:
— Данные официальной швейцарской службы предупреждения о граде VKF.
- Идеально подходит для автоматического складывания навесов и солнцезащитных штор в случае опасности града.
- **Доступно только в Швейцарии** (используется сервис VKF)
- Очень простая конфигурация

**Примечание:** Адаптер называется `hagelschutz-vkf`, потому что исходное название `hagelschutz` уже занято.

🔗 [Страница адаптера на GitHub](https://github.com/UncleSamSwiss/ioBroker.hagelschutz-vkf)

### IoBroker.foxesscloud
<img width="100" height="100" src="https://raw.githubusercontent.com/inventwo/ioBroker.foxesscloud/master/admin/foxesscloud.png" />

**Категория:** Энергетика / Солнечная энергия **Разработчик:** inventwo

Этот адаптер получает данные в режиме реального времени от солнечных инверторов FoxESS через облачный API FoxESS. Это позволяет удобно интегрировать солнечные энергетические системы, системы хранения энергии и подключение к сети в ioBroker.

FoxESS — широко известный производитель солнечных инверторов, которые также используются на солнечных электростанциях **Enpal**.

#### Зарегистрированные данные:
- `pvPower` – Текущая выработка электроэнергии фотоэлектрическими системами (кВт)
- `generationPower` – Суммарная выходная мощность (кВт)
- `нагрузка` – Текущее потребление (кВт)
- `gridConsumption` – Потребление электроэнергии из сети (кВт)
- `feedinPower` – Подача электроэнергии в сеть (кВт)
- `soc` – уровень заряда батареи (%)
- `batCharge` / `batDischarge` – Мощность зарядки/разрядки аккумулятора (кВт)
- Состояние подключения

#### Особые функции:
- До **1440 вызовов API в день** (1 вызов в минуту)
- Поддерживает системы FoxESS и Enpal.
- Идеально подходит для управления энергопотреблением, анализа потребления и автоматизации.
- Четкая структура данных

🔗 [Страница адаптера на GitHub](https://github.com/inventwo/ioBroker.foxesscloud)

### IoBroker.apsystems-ez1
<img width="100" height="100" src="https://raw.githubusercontent.com/zhihaining/ioBroker.apsystems-ez1/master/admin/apsystems-ez1.png" />

**Категория:** Энергетика / Солнечная энергия **Разработчик:** zhihaining

Этот адаптер интегрирует **микроинверторы APsystems EZ1** через локальный HTTP API устройства (порт 8050) – без облачных сервисов и внешних зависимостей.

Микроинверторы APsystems EZ1 — это компактные устройства, которые подключаются непосредственно к отдельным солнечным модулям и особенно подходят для **балконных электростанций** и небольших фотоэлектрических систем.

#### Поддерживаемые функции:
- Показатели производительности и энергопотребления в реальном времени (по каналам и в сумме)
- Информация об устройстве (прошивка, SSID, IP-адрес)
- Состояния тревожных сигналов запроса
- Установите **MaxPower** (ограничьте выходную мощность)
- Переключите устройство **вкл/выкл**
- Несколько устройств в одном экземпляре

#### Особые функции:
- Полностью **локальное** решение, не зависящее от облака.
- Логика тайм-аута HTTP и автоматической повторной попытки
- Дополнительное уведомление по электронной почте при повторных ошибках.
- Включает в себя **виджет VIS2** для обзора и управления.

🔗 [Страница адаптера на GitHub](https://github.com/zhihaining/ioBroker.apsystems-ez1)

### IoBroker.solectrus-influxdb
<img width="100" height="100" src="https://raw.githubusercontent.com/patricknitsch/ioBroker.solectrus-influxdb/master/admin/solectrus-influxdb.png" />

**Категория:** Энергетика / Базы данных **Разработчик:** patricknitsch

Этот адаптер записывает выбранные состояния ioBroker в базу данных **InfluxDB 2.x**, дополненную мощным механизмом формул для получения производных значений.

Он был специально разработан для систем мониторинга энергии (фотоэлектрические системы, аккумуляторные батареи, тепловые насосы, настенные зарядные устройства, учет потребления электроэнергии из сети).

#### Основные функции:
- **Сопоставление датчиков:** Настройте любые состояния ioBroker как измерения и поля InfluxDB (int, float, bool, string)
- **Надежный буфер:** Постоянный буфер записи (до 100 000 точек), сохраняется даже при сбоях InfluxDB.
- **Формулировочный механизм:** Расчетные значения на основе нескольких источников (например, коэффициент собственного потребления, коэффициент собственного покрытия)
- **Режим конечного автомата:** Строковые или логические состояния, полученные из условий, основанных на правилах.
- **Визуальный конструктор формул:** Перетаскивание, предварительный просмотр в реальном времени, всплывающие подсказки для операторов.
- Группировка папок для вычисляемых значений

#### Особые функции:
- Специально разработан для мониторинга энергопотребления с помощью **SOLECTRUS**
- Устойчивость к сбоям базы данных благодаря постоянной буферизации.
- Механизм формул позволяет выполнять сложные вычисления без внешнего адаптера скриптов.

🔗 [Страница адаптера на GitHub](https://github.com/patricknitsch/ioBroker.solectrus-influxdb)

### IoBroker.grohe-smarthome
<img width="100" height="100" src="https://raw.githubusercontent.com/patricknitsch/ioBroker.grohe-smarthome/master/admin/grohe-smarthome.png" />

**Категория:** Учёт потребления / Вода **Разработчик:** patricknitsch

Этот адаптер подключает ioBroker к **облаку Grohe Smarthome / Ondus** и предоставляет устройства мониторинга воды Grohe, как указано в описании ioBroker.

Компания Grohe предлагает интеллектуальные решения для управления водными ресурсами в умном доме, которые могут обнаруживать утечки, контролировать потребление воды и автоматически отключать воду при необходимости.

#### Поддерживаемые устройства:
- **Grohe Sense** (Тип 101) – Датчик влажности и температуры для обнаружения утечек
- **Grohe Sense Guard** (Тип 103) – Датчик водопроводной трубы с моторизованным запорным клапаном
- **Grohe Blue Home** (Тип 104) – Фильтрованный кран для домашнего использования
- **Grohe Blue Professional** (Тип 105) – Фильтрующий смеситель (профессиональная версия)

#### Поддерживаемые данные и функции:
- Показания температуры и влажности (датчик)
- Измерение расхода воды, значений давления и обнаружение утечек (Sense Guard)
- Управление клапаном (открытие/закрытие)
- Статистика потребления
- Сообщения об ошибках и тревогах

#### Особые функции:
- **Аутентификация OIDC/Keycloak** – Токены обновления хранятся в зашифрованном виде.
- Настраиваемый интервал запроса
- Полный комплект документов на немецком и английском языках.
- Идеально подходит для автоматизации защиты от протечек в умных домах.

🔗 [Страница адаптера на GitHub](https://github.com/patricknitsch/ioBroker.grohe-smarthome)

### IoBroker.meteoswiss
<img width="100" height="100" src="https://raw.githubusercontent.com/deMynchi/ioBroker.meteoswiss/master/admin/meteoswiss.png" />

**Категория:** Погода **Разработчик:** deMynchi

Адаптер предоставляет ioBroker бесплатные данные о погоде от **Федерального управления метеорологии и климатологии MeteoSwiss**, без необходимости ввода ключа API.

MeteoSwiss — это официальная метеорологическая служба Швейцарии, предоставляющая высокоточные метеорологические данные и предупреждения, адаптированные к швейцарскому рельефу.

#### Поддерживаемые данные:
- Текущая погода и краткосрочные прогнозы
- **Прогнозы погоды** с интервалом в 3 часа (включая агрегированные минимальные/максимальные/средние значения)
- **Предупреждения о неблагоприятных погодных условиях** с указанием уровня опасности: град, буря, дождь, снег и т. д.
- Автоматическое обновление каждые 10 минут (интервал подстраивается под источник).
— Четкое разграничение: значения `null` используются для неизвестных данных вместо null.

#### Особые функции:
- **Ключ API** не требуется, можно использовать напрямую.
- Поддерживает одновременную поддержку нескольких категорий предупреждений (отображает наиболее важную).
- Идеально подходит для пользователей из Швейцарии, которым необходима информация о погоде, привязанная к конкретному местоположению.
- Возможна автоматизация на основе предупреждений о неблагоприятных погодных условиях (например, автоматизация работы рольставней).

🔗 [Страница адаптера на GitHub](https://github.com/deMynchi/ioBroker.meteoswiss)

### IoBroker.autodarts
<img width="100" height="100" src="https://raw.githubusercontent.com/inventwo/ioBroker.autodarts/master/admin/autodarts.svg" />

**Категория:** Развлечения / Умный дом **Разработчик:** inventwo

Этот адаптер подключает ioBroker к **Autodarts Board Manager** — локально работающему серверу популярной системы электронных дартс-мишеней.

Autodarts позволяет превратить классические дартс-мишени в интеллектуальные системы автоматического распознавания с помощью камеры и программного обеспечения.

Этот адаптер позволяет интегрировать игровые события непосредственно в автоматизацию ioBroker: загораются индикаторы при попадании в цель, объявляется следующий выстрел через синтезатор речи или записывается статистика.

#### Зарегистрированные данные:
- Счет и текущая запись (`visit.score`, `throw.current`)
- Специальные эффекты попадания: Меткий выстрел, Тройной выстрел, Двойной выстрел, Промах
- Состояние доски (цвет светофора: зеленый = бросок, желтый = убрать дротики, красный = офлайн)
- Аппаратное обеспечение: Управление подсветкой платы и питанием.
- Информация о системе: версия программного обеспечения, процессор, конфигурация камеры.
- Поддержка синтеза речи для игровых анонсов

#### Особые функции:
- Подключение к локальному менеджеру доски Autodarts (стандартный порт `3180`)
- Отсутствие зависимости от облака – полностью локальное решение.
- Идеально подходит для автоматизации игровых событий (свет, звук, сцены).
- Действия, основанные на спусковом механизме, при попадании в центр мишени, тройном, двойном или любом другом броске дротика.

🔗 [Страница адаптера на GitHub](https://github.com/inventwo/ioBroker.autodarts)

### IoBroker.zwavews
<img width="100" height="100" src="https://raw.githubusercontent.com/arteck/ioBroker.zwavews/master/admin/zwavews.png" />

**Категория:** Протоколы / Умный дом **Разработчик:** arteck

Адаптер **ioBroker.zwavews** является современным преемником предыдущего адаптера `zwave2` и устанавливает соединение с устройствами Z-Wave через экземпляр **zwave-js-ui** по протоколу WebSocket.

**Важная информация:** Предыдущий адаптер `zwave2` был удален из репозитория `stable` в феврале 2026 года. Пользователям рекомендуется перейти на `zwavews`.

**Z-Wave** — это проверенный беспроводной протокол, специально разработанный для приложений умного дома.

Он работает в диапазоне частот ниже ГГц (868 МГц в Европе), обеспечивая надежную, бесперебойную альтернативу Wi-Fi и Zigbee — с ячеистой сетью и низким энергопотреблением.

#### Поддерживаемые функции:
- Интеграция с **zwave-js-ui** через WebSocket API Home Assistant.
- Автоматическое создание всех объектов устройств Z-Wave в виде состояний ioBroker.
- Установите значение `info.sendMessageAllowed` для контролируемого управления сообщениями после запуска.
- Простая миграция: скопируйте файл кэша из `zwave2` и активируйте устройства с питанием от батарей всего один раз.

#### Миграция `zwave2`:
1. Скопируйте файл кэша со старого адаптера в новый `zwave-js-ui`.
2. Активируйте устройства, работающие от батареек, один раз (нажмите кнопку).
3. Настройте и запустите адаптер `zwavews`.

🔗 [Страница адаптера на GitHub](https://github.com/arteck/ioBroker.zwavews)

Компания Mitsubishi Electric входит в число ведущих производителей сплит-систем кондиционирования воздуха и тепловых насосов для жилых и коммерческих помещений. Интерфейс **WLAN** позволяет управлять системой локально через домашнюю сеть.

Адаптер обеспечивает **полностью локальную интеграцию** устройства **SmartFriends Box** в ioBroker — полностью без подключения к облаку.

SmartFriends Box — это шлюз для умного дома, поддерживающий устройства от нескольких известных производителей: Schellenberg, ABUS, Paulmann, Steinel и других совместимых брендов.

## Примечание: `zwave2` удалено
Предыдущий адаптер **ioBroker.zwave2** был удален из репозитория `stable` в феврале 2026 года.

Его официальным преемником является новый адаптер **ioBroker.zwavews**, основанный на `zwave-js-ui` и его WebSocket API.

**Пользователям `zwave2` рекомендуется как можно скорее перейти на `zwavews`.** Переход возможен с минимальными затратами усилий – подробности можно найти в файле README нового адаптера.
