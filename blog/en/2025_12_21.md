### IoBroker.teltonika
<img width="100" height="100" src="https://raw.githubusercontent.com/ioBroker/ioBroker.teltonika/refs/heads/main/admin/teltonika.svg" />

**Category:** Network **Developer:** bluefox

This adapter enables the connection of **Teltonika routers** to ioBroker via efficient **MQTT communication**.

Teltonika routers are frequently used in professional and semi-professional environments, for example, for **LTE/5G backups**, **site networking**, **remote maintenance**, or **edge IoT applications**.

This adapter allows the status and telemetry data provided by the router to be seamlessly integrated into **ioBroker dashboards**, **monitoring setups**, or **automations**.

The adapter supports a wide variety of router models, including series such as **RUT**, **TRB**, **OTD**, **RUTM** and **RUTC**, and provides extensive device data as states.

#### Data points captured (model-dependent):
- Temperature (`RUT2`, `RUT9`, `RUTX`, `RUT3`, `RUT1`, `TRB2`, `TRB5`, `OTD`, `RUTM`, `RUTC`)
- Signal strength & mobile network provider
- Network status & connection type (2G/3G/4G)
- WAN IP address
- Uptime
- Device name
- Digital inputs (`RUT9`: Input 1/2)
- Analog inputs (`RUT9`, `TRB2`, `TRB141`)
- Pin status (Pin 2, 3, 4 ‚Äì model dependent)

#### Special features:
- **MQTT-based** for fast, resource-efficient communication
- Uses the dedicated port **1885** to avoid conflicts with other MQTT instances
- **Easy start:** Simply activate MQTT in the router and point it to the ioBroker broker.
- **Tested with RUTC series devices**, compatible with all devices that support MQTT
- **Sentry integration** for automatic error detection (from js-controller 3.0 onwards)
- Ideal for network monitoring, logging, alerts and automation scenarios

## Adapter Updates and Features
### IoBroker.ping ‚Äì new feature: TCP port check
<img width="100" height="100" src="https://raw.githubusercontent.com/ioBroker/ioBroker.ping/master/admin/ping.png?raw=true" />

The popular **ioBroker.ping** adapter has been enhanced and can now check **TCP ports** in addition to classic ICMP pings.

This allows you to determine not only whether a device is reachable, but also whether a specific **service** is running on the target system ‚Äì for example, a web server, an MQTT broker, or an SSH service.

#### What is the benefit of TCP port checking?
With this new feature, the adapter can now:

- Check if a specific port is **open** or **closed**
- To determine whether a service is **available** or **out of service**
- Make network-dependent automations more reliable
- Provides more precise status updates than a classic ping

Example: A host responds to ping, but port 1883 (MQTT) is closed ‚Üí indication of a service problem.

#### Typical use cases:
- Check if the **ioBroker** port of a remote system is reachable
- Monitoring of web servers (port 80/443)
- Monitoring of SSH (Port 22)
- Availability checks for databases, gateways, smart home devices
- Triggering actions when services fail

#### Special features:
- Works in addition to the normal ping ‚Äì both can be used simultaneously.
- Very low system load
- Simple configuration: Enter IP/hostname + port
- Ideal for home networks, servers, NAS and smart home infrastructures

üîó [GitHub](https://github.com/ioBroker/ioBroker.ping)
