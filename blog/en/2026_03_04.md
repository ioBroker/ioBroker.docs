# IoBroker Monthly Review – January/February 2026
The new year has kicked off with a bang in ioBroker! In the first two months of 2026, numerous new adapters were added to the `latest` repository, and some of them have already been upgraded to the `stable` version.

There's also important news for Z-Wave users: the previous `zwave2` adapter has been replaced by a more modern solution.

## New adapters in `latest`
These adapters welcome testers, bug reports, and suggestions for improvement before they may be released as stable versions.

### IoBroker.hagelschutz-vkf
<img width="100" height="100" src="https://raw.githubusercontent.com/UncleSamSwiss/ioBroker.hagelschutz-vkf/master/admin/hagelschutz-vkf.jpg" />

**Category:** Weather / Building Protection **Developer:** UncleSamSwiss

This adapter integrates the service **hagelschutz-einfach-automatisch.ch** from the **Association of Cantonal Building Insurers (VKF)** into ioBroker.

The service automatically warns of hail events, allowing awnings, roller shutters, and other sensitive outdoor installations to be retracted in time.

#### Special features:
- Data from the official VKF hail warning service of Switzerland
- Ideal for automatically retracting awnings and sunshades in case of hail danger
- **Only available in Switzerland** (uses the VKF service)
- Very simple configuration

**Note:** The adapter is called `hagelschutz-vkf` because the original name `hagelschutz` was already taken.

🔗 [GitHub page of the adapter](https://github.com/UncleSamSwiss/ioBroker.hagelschutz-vkf)

### IoBroker.foxesscloud
<img width="100" height="100" src="https://raw.githubusercontent.com/inventwo/ioBroker.foxesscloud/master/admin/foxesscloud.png" />

**Category:** Energy / Solar **Developer:** inventwo

This adapter retrieves real-time data from **FoxESS solar inverters** via the **FoxESS Cloud API**. This allows solar power systems, battery storage, and grid feed-in to be conveniently integrated into ioBroker.

FoxESS is a widely used manufacturer of solar inverters, which are also used in **Enpal** solar power plants.

#### Data points recorded:
- `pvPower` – Current PV generation (kW)
- `generationPower` – Total output power (kW)
- `load` – Current consumption (kW)
- `gridConsumption` – Grid consumption (kW)
- `feedinPower` – Grid feed-in (kW)
- `soc` – battery charge level (%)
- `batCharge` / `batDischarge` – Battery charging/discharging power (kW)
- Connection status

#### Special features:
- Up to **1440 API calls per day** (1 call per minute)
- Supports FoxESS and Enpal systems
- Ideal for energy management, consumption analysis and automation
- Clear data point structure

🔗 [GitHub page of the adapter](https://github.com/inventwo/ioBroker.foxesscloud)

### IoBroker.apsystems-ez1
<img width="100" height="100" src="https://raw.githubusercontent.com/zhihaining/ioBroker.apsystems-ez1/master/admin/apsystems-ez1.png" />

**Category:** Energy / Solar **Developer:** zhihaining

This adapter integrates **APsystems EZ1 microinverters** via the device's local HTTP API (port 8050) – without cloud, without external dependencies.

APsystems EZ1 are compact microinverters that are connected directly to individual solar modules and are particularly suitable for **balcony power plants** and small PV systems.

#### Supported functions:
- Real-time performance and energy values (per channel and total)
- Device information (firmware, SSID, IP address)
- Query alarm states
- Set **MaxPower** (limit the output power)
- Switch device **On/Off**
- Multiple devices in one instance

#### Special features:
- Completely **local**, no cloud dependency
- HTTP timeout and automatic retry logic
- Optional **email notification** for repeated errors
- Includes a **VIS2 widget** for overview and control

🔗 [GitHub page of the adapter](https://github.com/zhihaining/ioBroker.apsystems-ez1)

### IoBroker.solectrus-influxdb
<img width="100" height="100" src="https://raw.githubusercontent.com/patricknitsch/ioBroker.solectrus-influxdb/master/admin/solectrus-influxdb.png" />

**Category:** Energy / Database **Developer:** patricknitsch

This adapter writes selected ioBroker states to an **InfluxDB 2.x** database, supplemented by a powerful formula engine for derived values.

It was specifically developed for energy monitoring systems (PV systems, battery storage, heat pumps, wallboxes, grid metering).

#### Main functions:
- **Sensor Mapping:** Configure any ioBroker states as InfluxDB measurements and fields (int, float, bool, string)
- **Reliable Buffer:** Persistent write buffer (up to 100,000 points), survives InfluxDB outages
- **Formula Engine:** Calculated values from multiple sources (e.g., self-consumption rate, self-coverage rate)
- **State Machine Mode:** String or Boolean states from rule-based conditions
- **Visual Formula Builder:** Drag-and-drop, live preview, operator tooltips
- Folder grouping for calculated values

#### Special features:
- Specifically designed for energy monitoring with **SOLECTRUS**
- Survives database failures through persistent buffering
- Formula engine enables complex calculations without an external script adapter

🔗 [GitHub page of the adapter](https://github.com/patricknitsch/ioBroker.solectrus-influxdb)

### IoBroker.grohe-smarthome
<img width="100" height="100" src="https://raw.githubusercontent.com/patricknitsch/ioBroker.grohe-smarthome/master/admin/grohe-smarthome.png" />

**Category:** Metering / Water **Developer:** patricknitsch

This adapter connects ioBroker to the **Grohe Smarthome / Ondus Cloud** and provides Grohe water monitoring devices as ioBroker states.

Grohe offers smart water management solutions for the smart home that can detect leaks, monitor water consumption and automatically shut off the water when necessary.

#### Supported devices:
- **Grohe Sense** (Type 101) – Humidity and temperature sensor for leak detection
- **Grohe Sense Guard** (Type 103) – Water pipe sensor with motorized shut-off valve
- **Grohe Blue Home** (Type 104) – Filtered tap for home use
- **Grohe Blue Professional** (Type 105) – Filtered faucet (professional version)

#### Supported Data & Functions:
- Temperature and humidity readings (Sense)
- Water flow, pressure values and leak detection (Sense Guard)
- Valve control (open/close)
- Consumption statistics
- Error and alarm messages

#### Special features:
- **OIDC/Keycloak authentication** – Refresh tokens stored encrypted
- Configurable query interval
- Complete documentation in German and English
- Ideal for leak protection automation in smart homes

🔗 [GitHub page of the adapter](https://github.com/patricknitsch/ioBroker.grohe-smarthome)

## New adapters in `stable`
### IoBroker.meteoswiss
<img width="100" height="100" src="https://raw.githubusercontent.com/deMynchi/ioBroker.meteoswiss/master/admin/meteoswiss.png" />

**Category:** Weather **Developer:** deMynchi

The adapter provides weather data from the **Federal Office for Meteorology and Climatology MeteoSwiss** for ioBroker, free of charge and without an API key.

MeteoSwiss is the official weather service of Switzerland and provides highly accurate weather data and warnings tailored to the Swiss topography.

#### Supported data:
- Current weather and short-term forecasts
- **Weather forecasts** in 3-hour intervals (including aggregated min/max/average values)
- **Severe weather warnings** with severity level information: hail, storm, rain, snow, etc.
- Automatic update every 10 minutes (interval adjusts to the source)
- Clear distinction: `null` values for unknown data instead of null.

#### Special features:
- **No API key** required, usable directly
- Supports multiple warning categories simultaneously (displays the most important one)
- Ideal for Swiss users with location-specific weather information
- Automations based on severe weather warnings are possible (e.g., retracting roller shutters)

🔗 [GitHub page of the adapter](https://github.com/deMynchi/ioBroker.meteoswiss)

### IoBroker.autodarts
<img width="100" height="100" src="https://raw.githubusercontent.com/inventwo/ioBroker.autodarts/master/admin/autodarts.svg" />

**Category:** Entertainment / Smart Home **Developer:** inventwo

This adapter connects ioBroker to the **Autodarts Board Manager** – the locally running server of the popular electronic dartboard system.

Autodarts allows you to transform classic dartboards into smart, auto-recognizing systems using a camera and software.

This adapter allows game events to be integrated directly into ioBroker automations: lights to light up when a bullseye is hit, the next shot to be announced via TTS, or statistics to be logged.

#### Data points recorded:
- Score & current recording (`visit.score`, `throw.current`)
- Special hit triggers: Bullseye, Triple, Double, Miss
- Board status (traffic light color: green = throw, yellow = remove darts, red = offline)
- Hardware: Board lighting & power supply controllable
- System information: Software version, CPU, camera configuration
- TTS support for game announcements

#### Special features:
- Connection to the local Autodarts Board Manager (standard port `3180`)
- No cloud dependency – completely local
- Ideal for automating game events (lights, sound, scenes)
- Trigger-based actions on the bullseye, triple, double, or any dart throw

🔗 [GitHub page of the adapter](https://github.com/inventwo/ioBroker.autodarts)

### IoBroker.zwavews
<img width="100" height="100" src="https://raw.githubusercontent.com/arteck/ioBroker.zwavews/master/admin/zwavews.png" />

**Category:** Protocols / Smart Home **Developer:** arteck

The **ioBroker.zwavews** adapter is the modern successor to the previous `zwave2` adapter and establishes the connection to Z-Wave devices via a **zwave-js-ui** instance via WebSocket.

**Important Information:** The previous `zwave2` adapter was removed from the `stable` repository in February 2026. Users are requested to migrate to `zwavews`.

**Z-Wave** is a proven wireless protocol specifically designed for smart home applications.

It operates in the sub-GHz range (868 MHz in Europe), providing a reliable, interference-free alternative to Wi-Fi and Zigbee – with mesh networking and low energy consumption.

#### Supported functions:
- Connection to **zwave-js-ui** via the Home Assistant WebSocket API
- Automatic creation of all Z-Wave device objects as ioBroker states
- State `info.sendMessageAllowed` for controlled message management after startup
- Easy migration: Copy the cache file from `zwave2` and wake up battery-powered devices once.

#### Migration of `zwave2`:
1. Copy the cache file from the old adapter to the new `zwave-js-ui`.
2. Wake up battery-operated devices once (press button)
3. Configure and start the `zwavews` adapter

🔗 [GitHub page of the adapter](https://github.com/arteck/ioBroker.zwavews)

## Note: `zwave2` removed
The previous **ioBroker.zwave2** adapter was removed from the `stable` repository in February 2026.

The new **ioBroker.zwavews** adapter, which uses `zwave-js-ui` and its WebSocket API, is now available as its official successor.

**Users of `zwave2` are requested to switch to `zwavews` as soon as possible.** The migration is possible with manageable effort – details can be found in the README of the new adapter.
